The Application & Window Classes

Now we turn the SDL3 code from the platform chapter into reusable engine classes. Window owns the SDL_Window and SDL's lifetime, while Application owns the validated EngineConfig, running state, and frame loop. This code is identical on Windows and macOS; only the renderer behind it changes.

What you'll learn
  • Use RAII to own an SDL_Window* and the SDL video subsystem.
  • Keep the application's running state in one place.
  • Handle close and high-DPI resize events through one SDL3 event loop.
  • Expose drawable pixel size correctly on Retina and scaled displays.
  • Load YAML once at startup and pass typed settings into the application.
  • Keep native graphics surfaces inside renderer backends.

A small Window owner

#pragma once
#include <string>

struct SDL_Window; // forward declaration keeps SDL headers out of this file

class Window {
public:
    Window() = default;
    ~Window();

    Window(const Window&) = delete;
    Window& operator=(const Window&) = delete;

    bool Create(const std::string& title, int width, int height,
                bool resizable);
    bool GetFramebufferSize(int& width, int& height) const;

    // Renderer initialization may use this opaque SDL handle.
    SDL_Window* NativeHandle() const { return m_handle; }

private:
    SDL_Window* m_handle = nullptr;
    bool m_sdlInitialized = false;
};

The public header knows only that SDL_Window is a type. Window creation, error reporting, and destruction stay in the implementation file that includes SDL3.

Implementing Window with SDL3

#include "Window.h"
#include <SDL3/SDL.h>
#include <iostream>

Window::~Window() {
    // The renderer must release its swap chain or SDL_MetalView first.
    if (m_handle) {
        SDL_DestroyWindow(m_handle);
        m_handle = nullptr;
    }
    if (m_sdlInitialized) {
        SDL_Quit();
        m_sdlInitialized = false;
    }
}

bool Window::Create(const std::string& title, int width, int height,
                    bool resizable) {
    if (!SDL_Init(SDL_INIT_VIDEO)) {
        std::cerr << "SDL initialization failed: "
                  << SDL_GetError() << '\n';
        return false;
    }
    m_sdlInitialized = true;

    SDL_WindowFlags flags = SDL_WINDOW_HIGH_PIXEL_DENSITY;
    if (resizable) flags |= SDL_WINDOW_RESIZABLE;

#if defined(__APPLE__)
    // The Metal backend will create an SDL_MetalView in the next lesson.
    flags |= SDL_WINDOW_METAL;
#endif

    m_handle = SDL_CreateWindow(title.c_str(), width, height, flags);
    if (!m_handle) {
        std::cerr << "Window creation failed: "
                  << SDL_GetError() << '\n';
        return false;
    }
    return true;
}

bool Window::GetFramebufferSize(int& width, int& height) const {
    return m_handle &&
           SDL_GetWindowSizeInPixels(m_handle, &width, &height);
}
SDL3 returns bool

In SDL3, SDL_Init and SDL_GetWindowSizeInPixels return true on success. Older SDL2 examples often compare initialization with 0; do not copy that convention into this project.

The Application owns the loop

The window owns the operating-system resource, but the application decides whether the game is still running. It also keeps the typed configuration loaded in the previous lesson. A close request changes m_running; game code never needs a Win32 message, Cocoa event, or YAML node.

#pragma once
#include "EngineConfig.h"
#include "Platform/Window.h"

class Application {
public:
    explicit Application(EngineConfig config);
    virtual ~Application() = default;
    int Run();

protected:
    virtual bool OnInitialize() { return true; }
    virtual void OnResize(int pixelWidth, int pixelHeight) {}
    virtual void OnUpdate(float deltaTime) {}
    virtual void OnRender() {}
    virtual void OnShutdown() {}

    void Quit() { m_running = false; }
    Window& GetWindow() { return m_window; }
    const EngineConfig& GetConfig() const { return m_config; }

private:
    EngineConfig m_config;
    Window m_window;           // destroyed after OnShutdown returns
    bool m_running = false;    // one owner for application lifetime
};
#include "Application.h"
#include <SDL3/SDL.h>
#include <chrono>
#include <utility>

Application::Application(EngineConfig config)
    : m_config(std::move(config)) {}

int Application::Run() {
    const WindowConfig& window = m_config.window;
    if (!m_window.Create(window.title, window.width, window.height,
                         window.resizable)) return 1;
    if (!OnInitialize()) {
        OnShutdown();
        return 1;
    }

    using Clock = std::chrono::steady_clock;
    auto previous = Clock::now();
    m_running = true;

    while (m_running) {
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_EVENT_QUIT ||
                event.type == SDL_EVENT_WINDOW_CLOSE_REQUESTED) {
                m_running = false;
            }

            if (event.type == SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED) {
                OnResize(event.window.data1, event.window.data2);
            }
        }

        if (!m_running) break;

        const auto now = Clock::now();
        const float dt = std::chrono::duration<float>(now - previous).count();
        previous = now;

        OnUpdate(dt);
        OnRender();
    }

    // GPU presentation objects are released here. Window's destructor runs
    // afterward, so no renderer can outlive its SDL surface.
    OnShutdown();
    return 0;
}
Use pixel-size events for renderer resize

SDL_EVENT_WINDOW_RESIZED reports window coordinates. Render targets need physical pixels, so use the shared, guaranteed SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED path on both platforms. Skip target recreation when either dimension is zero.

Using the framework

The source file containing main includes SDL_main.h exactly once. It loads configuration before creating the application, reports a bad YAML file cleanly, and passes only typed C++ into the engine.

#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>
#include "Core/Application.h"
#include <exception>
#include <iostream>
#include <utility>

class Game : public Application {
public:
    explicit Game(EngineConfig config)
        : Application(std::move(config)) {}

protected:
    bool OnInitialize() override { return true; }
    void OnUpdate(float dt) override { m_time += dt; }
    void OnRender() override { /* renderer comes next */ }

private:
    float m_time = 0.0f;
};

int main(int argc, char* argv[]) {
    try {
        Game game{LoadStartupConfig()};
        return game.Run();
    } catch (const std::exception& error) {
        std::cerr << "Startup failed: " << error.what() << '\n';
        return 1;
    }
}

Recap
  • Window owns SDL_Window and balances SDL_Init with SDL_Quit.
  • main loads YAML once; Application and Window consume validated typed settings.
  • Application owns m_running, drains SDL events, and calls portable update/render hooks.
  • SDL_WINDOW_HIGH_PIXEL_DENSITY plus pixel-size events keeps render targets sharp on Retina displays.
  • std::chrono::steady_clock produces frame-rate-independent delta time.
  • Renderer shutdown happens before Window destruction, which is required by both DXGI and SDL's Metal view.
