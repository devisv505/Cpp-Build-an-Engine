Abstracting the Renderer

SDL3 gave the engine one window and event loop, but it does not turn Direct3D commands into Metal commands. We still need two renderer implementations. The important improvement is that the rest of the engine sees one Renderer contract and passes the same SDL_Window* to either backend.

What you'll learn
  • Why SDL3 unifies the platform layer but not the graphics APIs.
  • How one Renderer::Initialize(SDL_Window*) seam serves both backends.
  • How Direct3D retrieves the SDL window's HWND without leaking Win32 into shared code.
  • How Metal uses SDL_MetalView and a borrowed CAMetalLayer.
  • How BeginFrame, EndFrame, and Resize express the shared frame lifecycle.
  • How the shared VSync setting maps to DXGI and CAMetalLayer.
  • Why renderer shutdown must happen before the SDL window is destroyed.

The shared renderer contract

The interface contains standard C++ data plus one opaque SDL window pointer. No Direct3D or Metal type reaches the scene, material, or gameplay layer.

#pragma once
#include <array>

struct SDL_Window;

class Renderer {
public:
    virtual ~Renderer() = default;

    virtual bool Initialize(SDL_Window* window, bool vsync) = 0;
    virtual void BeginFrame(const std::array<float, 4>& clearColor) = 0;
    virtual void EndFrame() = 0;
    virtual void Resize(int pixelWidth, int pixelHeight) = 0;
};
What is really shared?

The scene traversal, camera, visible-object list, clear color, and calls to BeginFrame/EndFrame are shared. Device creation, shaders, buffers, draw encoding, and presentation remain native backend work. That is the normal shape of a cross-platform engine.

Windows: obtain the HWND from SDL3

DXGI still requires a Win32 HWND when it creates the swap chain. SDL3 stores that pointer as a read-only property on its window. Only the Windows renderer reads it.

#pragma once
#include "Renderer.h"
#include <d3d11.h>
#include <wrl/client.h>

class D3D11Renderer final : public Renderer {
public:
    ~D3D11Renderer() override;
    bool Initialize(SDL_Window* window, bool vsync) override;
    void BeginFrame(const std::array<float, 4>& clearColor) override;
    void EndFrame() override;
    void Resize(int pixelWidth, int pixelHeight) override;

    ID3D11Device* Device() const { return m_device.Get(); }
    ID3D11DeviceContext* Context() const { return m_context.Get(); }

private:
    bool CreateRenderTargets(int pixelWidth, int pixelHeight);

    template <typename T>
    using ComPtr = Microsoft::WRL::ComPtr<T>;

    ComPtr<ID3D11Device> m_device;
    ComPtr<ID3D11DeviceContext> m_context;
    ComPtr<IDXGISwapChain> m_swapChain;
    ComPtr<ID3D11RenderTargetView> m_renderTargetView;
    ComPtr<ID3D11DepthStencilView> m_depthStencilView;
    D3D11_VIEWPORT m_viewport{};
    bool m_vsync = true;
};
#include "D3D11Renderer.h"
#include <SDL3/SDL.h>
#include <windows.h>

D3D11Renderer::~D3D11Renderer() {
    if (m_context) {
        m_context->ClearState();
        m_context->Flush();
    }
}

bool D3D11Renderer::Initialize(SDL_Window* window, bool vsync) {
    if (!window) return false;
    m_vsync = vsync;

    const SDL_PropertiesID properties = SDL_GetWindowProperties(window);
    HWND hwnd = static_cast<HWND>(SDL_GetPointerProperty(
        properties, SDL_PROP_WINDOW_WIN32_HWND_POINTER, nullptr));

    int pixelWidth = 0;
    int pixelHeight = 0;
    if (!hwnd ||
        !SDL_GetWindowSizeInPixels(window, &pixelWidth, &pixelHeight) ||
        pixelWidth <= 0 || pixelHeight <= 0) {
        return false;
    }

    DXGI_SWAP_CHAIN_DESC desc{};
    desc.BufferCount = 2;
    desc.BufferDesc.Width = static_cast<UINT>(pixelWidth);
    desc.BufferDesc.Height = static_cast<UINT>(pixelHeight);
    desc.BufferDesc.Format = DXGI_FORMAT_R8G8B8A8_UNORM;
    desc.BufferUsage = DXGI_USAGE_RENDER_TARGET_OUTPUT;
    desc.OutputWindow = hwnd;
    desc.SampleDesc.Count = 1;
    desc.Windowed = TRUE;
    desc.SwapEffect = DXGI_SWAP_EFFECT_FLIP_DISCARD;

    UINT flags = 0;
#if !defined(NDEBUG)
    flags |= D3D11_CREATE_DEVICE_DEBUG;
#endif

    const HRESULT hr = D3D11CreateDeviceAndSwapChain(
        nullptr,
        D3D_DRIVER_TYPE_HARDWARE,
        nullptr,
        flags,
        nullptr, 0,
        D3D11_SDK_VERSION,
        &desc,
        m_swapChain.GetAddressOf(),
        m_device.GetAddressOf(),
        nullptr,
        m_context.GetAddressOf());

    return SUCCEEDED(hr) &&
           CreateRenderTargets(pixelWidth, pixelHeight);
}

SDL_GetPointerProperty returns a borrowed pointer. Do not destroy the HWND; SDL's Window owner will destroy the native window after the renderer releases its swap chain.

Create the color and depth targets

bool D3D11Renderer::CreateRenderTargets(int width, int height) {
    ComPtr<ID3D11Texture2D> backBuffer;
    HRESULT hr = m_swapChain->GetBuffer(
        0, __uuidof(ID3D11Texture2D),
        reinterpret_cast<void**>(backBuffer.GetAddressOf()));
    if (FAILED(hr)) return false;

    hr = m_device->CreateRenderTargetView(
        backBuffer.Get(), nullptr, m_renderTargetView.GetAddressOf());
    if (FAILED(hr)) return false;

    D3D11_TEXTURE2D_DESC depthDesc{};
    depthDesc.Width = static_cast<UINT>(width);
    depthDesc.Height = static_cast<UINT>(height);
    depthDesc.MipLevels = 1;
    depthDesc.ArraySize = 1;
    depthDesc.Format = DXGI_FORMAT_D24_UNORM_S8_UINT;
    depthDesc.SampleDesc.Count = 1;
    depthDesc.Usage = D3D11_USAGE_DEFAULT;
    depthDesc.BindFlags = D3D11_BIND_DEPTH_STENCIL;

    ComPtr<ID3D11Texture2D> depthTexture;
    hr = m_device->CreateTexture2D(
        &depthDesc, nullptr, depthTexture.GetAddressOf());
    if (FAILED(hr)) return false;

    hr = m_device->CreateDepthStencilView(
        depthTexture.Get(), nullptr, m_depthStencilView.GetAddressOf());
    if (FAILED(hr)) return false;

    m_viewport.Width = static_cast<float>(width);
    m_viewport.Height = static_cast<float>(height);
    m_viewport.MinDepth = 0.0f;
    m_viewport.MaxDepth = 1.0f;
    return true;
}

Begin, present, and resize

void D3D11Renderer::BeginFrame(
    const std::array<float, 4>& clearColor) {
    m_context->OMSetRenderTargets(
        1, m_renderTargetView.GetAddressOf(), m_depthStencilView.Get());
    m_context->ClearRenderTargetView(
        m_renderTargetView.Get(), clearColor.data());
    m_context->ClearDepthStencilView(
        m_depthStencilView.Get(),
        D3D11_CLEAR_DEPTH | D3D11_CLEAR_STENCIL, 1.0f, 0);
    m_context->RSSetViewports(1, &m_viewport);
}

void D3D11Renderer::EndFrame() {
    m_swapChain->Present(m_vsync ? 1 : 0, 0);
}

void D3D11Renderer::Resize(int width, int height) {
    if (!m_swapChain || width <= 0 || height <= 0) return;

    m_context->OMSetRenderTargets(0, nullptr, nullptr);
    m_renderTargetView.Reset();
    m_depthStencilView.Reset();

    const HRESULT hr = m_swapChain->ResizeBuffers(
        0, static_cast<UINT>(width), static_cast<UINT>(height),
        DXGI_FORMAT_UNKNOWN, 0);
    if (SUCCEEDED(hr)) CreateRenderTargets(width, height);
}
Release views before ResizeBuffers

The render-target and depth views reference the old swap-chain textures. Unbind and reset them before resizing, then recreate the views and viewport with the new physical pixel dimensions.

macOS: let SDL3 create the Metal view

The previous graphics lessons introduced the same objects directly. Inside the engine, SDL3 can create and attach the Metal-backed view for us. SDL_Metal_GetLayer returns its CAMetalLayer; our renderer configures and uses that layer but does not own or release it.

#pragma once
#include "Renderer.h"
#include <SDL3/SDL_metal.h>

namespace NS { class AutoreleasePool; }
namespace MTL {
    class Device;
    class CommandQueue;
    class CommandBuffer;
    class RenderCommandEncoder;
}
namespace CA { class MetalLayer; class MetalDrawable; }

class MetalRenderer final : public Renderer {
public:
    ~MetalRenderer() override;

    bool Initialize(SDL_Window* window, bool vsync) override;
    void BeginFrame(const std::array<float, 4>& clearColor) override;
    void EndFrame() override;
    void Resize(int pixelWidth, int pixelHeight) override;

private:
    void Shutdown();

    SDL_MetalView m_metalView = nullptr; // owned: destroy with SDL
    CA::MetalLayer* m_layer = nullptr;   // borrowed from m_metalView
    MTL::Device* m_device = nullptr;
    MTL::CommandQueue* m_commandQueue = nullptr;

    NS::AutoreleasePool* m_framePool = nullptr;
    CA::MetalDrawable* m_drawable = nullptr;
    MTL::CommandBuffer* m_commands = nullptr;
    MTL::RenderCommandEncoder* m_encoder = nullptr;
    bool m_frameReady = false;
};
#include "MetalRenderer.h"
#include <SDL3/SDL.h>
#include <Foundation/Foundation.hpp>
#include <Metal/Metal.hpp>
#include <QuartzCore/QuartzCore.hpp>

MetalRenderer::~MetalRenderer() {
    Shutdown();
}

bool MetalRenderer::Initialize(SDL_Window* window, bool vsync) {
    if (!window) return false;

    m_device = MTL::CreateSystemDefaultDevice();
    if (!m_device) return false;

    m_commandQueue = m_device->newCommandQueue();
    if (!m_commandQueue) {
        Shutdown();
        return false;
    }

    m_metalView = SDL_Metal_CreateView(window);
    if (!m_metalView) {
        Shutdown();
        return false;
    }

    // SDL_MetalView owns this layer. Borrow it—do not retain or release it.
    m_layer = static_cast<CA::MetalLayer*>(
        SDL_Metal_GetLayer(m_metalView));
    if (!m_layer) {
        Shutdown();
        return false;
    }

    m_layer->setDevice(m_device);
    m_layer->setPixelFormat(MTL::PixelFormatBGRA8Unorm);
    m_layer->setFramebufferOnly(true);
    m_layer->setDisplaySyncEnabled(vsync);

    int pixelWidth = 0;
    int pixelHeight = 0;
    if (!SDL_GetWindowSizeInPixels(window, &pixelWidth, &pixelHeight)) {
        Shutdown();
        return false;
    }
    Resize(pixelWidth, pixelHeight);
    return true;
}

void MetalRenderer::Resize(int width, int height) {
    if (!m_layer || width <= 0 || height <= 0) return;

    // SDL3 automatically updates its Metal layer's drawable size.
    // Later, rebuild Metal depth targets and viewport state here.

void MetalRenderer::Shutdown() {
    if (m_framePool) {
        m_framePool->release();
        m_framePool = nullptr;
    }
    m_encoder = nullptr;
    m_commands = nullptr;
    m_drawable = nullptr;
    m_frameReady = false;

    // The layer becomes invalid when its owning SDL_MetalView is destroyed.
    m_layer = nullptr;
    if (m_metalView) {
        SDL_Metal_DestroyView(m_metalView);
        m_metalView = nullptr;
    }

    if (m_commandQueue) {
        m_commandQueue->release();
        m_commandQueue = nullptr;
    }
    if (m_device) {
        m_device->release();
        m_device = nullptr;
    }
}
Do not release the borrowed layer

SDL_Metal_CreateView owns the view and its CAMetalLayer. Store the layer only while that view exists. Call SDL_Metal_DestroyView exactly once during renderer shutdown and always before SDL_DestroyWindow.

Encode the same frame lifecycle

void MetalRenderer::BeginFrame(
    const std::array<float, 4>& color) {
    // A failed frame must not reuse pointers from the previous frame.
    m_frameReady = false;
    m_encoder = nullptr;
    m_commands = nullptr;
    m_drawable = nullptr;
    if (m_framePool) {
        m_framePool->release();
        m_framePool = nullptr;
    }

    if (!m_layer || !m_commandQueue) return;

    m_framePool = NS::AutoreleasePool::alloc()->init();
    m_drawable = m_layer->nextDrawable();
    m_commands = m_commandQueue->commandBuffer();
    if (!m_drawable || !m_commands) {
        m_drawable = nullptr;
        m_commands = nullptr;
        m_framePool->release();
        m_framePool = nullptr;
        return; // normal while minimized or temporarily unavailable
    }

    auto* pass = MTL::RenderPassDescriptor::renderPassDescriptor();
    auto* target = pass->colorAttachments()->object(0);
    target->setTexture(m_drawable->texture());
    target->setLoadAction(MTL::LoadActionClear);
    target->setStoreAction(MTL::StoreActionStore);
    target->setClearColor(MTL::ClearColor(
        color[0], color[1], color[2], color[3]));

    m_encoder = m_commands->renderCommandEncoder(pass);
    if (!m_encoder) {
        m_commands = nullptr;
        m_drawable = nullptr;
        m_framePool->release();
        m_framePool = nullptr;
        return;
    }
    m_frameReady = true;
}

void MetalRenderer::EndFrame() {
    if (m_frameReady) {
        m_encoder->endEncoding();
        m_commands->presentDrawable(m_drawable);
        m_commands->commit();
    }

    m_frameReady = false;
    m_encoder = nullptr;
    m_commands = nullptr;
    m_drawable = nullptr;
    if (m_framePool) {
        m_framePool->release();
        m_framePool = nullptr;
    }
}

A missing drawable is normal while the window is minimized or temporarily unavailable. BeginFrame clears every per-frame pointer first and leaves m_frameReady false; EndFrame then becomes a safe no-op instead of presenting a drawable left over from an earlier frame.

The implementation macros for metal-cpp remain in the single MetalCpp.cpp translation unit created in Lesson 39. Do not define NS_PRIVATE_IMPLEMENTATION, CA_PRIVATE_IMPLEMENTATION, or MTL_PRIVATE_IMPLEMENTATION again in MetalRenderer.cpp. SDL supplies the Metal view, so this path needs no Objective-C++ or hand-written Cocoa bridge.

Select the backend in one factory

A small factory is the only shared source file with a platform branch. It constructs the backend CMake compiled and returns the common base type.

#pragma once
#include <memory>

class Renderer;
std::unique_ptr<Renderer> CreateRenderer();
#include "RendererFactory.h"

#if defined(_WIN32)
    #include "D3D11Renderer.h"
#elif defined(__APPLE__)
    #include "MetalRenderer.h"
#endif

std::unique_ptr<Renderer> CreateRenderer() {
#if defined(_WIN32)
    return std::make_unique<D3D11Renderer>();
#elif defined(__APPLE__)
    return std::make_unique<MetalRenderer>();
#else
    return nullptr;
#endif
}

Use either backend from the same Application

The application hooks from the previous lesson now contain neither a graphics API nor an operating-system branch. They also receive the same validated YAML configuration on both platforms, so shared render settings do not need backend-specific files.

#include "Core/Application.h"
#include "Renderer/Renderer.h"
#include "Renderer/RendererFactory.h"
#include <memory>
#include <utility>

class Game final : public Application {
public:
    explicit Game(EngineConfig config)
        : Application(std::move(config)) {}

protected:
    bool OnInitialize() override {
        m_renderer = CreateRenderer();
        const bool vsync = GetConfig().renderer.vsync;
        return m_renderer &&
               m_renderer->Initialize(GetWindow().NativeHandle(), vsync);
    }

    void OnResize(int pixelWidth, int pixelHeight) override {
        if (m_renderer) m_renderer->Resize(pixelWidth, pixelHeight);
    }

    void OnRender() override {
        m_renderer->BeginFrame(GetConfig().renderer.clearColor);
        // Later: submit visible meshes here.
        m_renderer->EndFrame();
    }

    void OnShutdown() override {
        // Metal destroys SDL_MetalView here, before Window's destructor.
        m_renderer.reset();
    }

private:
    std::unique_ptr<Renderer> m_renderer;
};

Both renderer settings now have an effect: the clear color feeds BeginFrame, while VSync chooses DXGI's presentation interval and Metal's display-synchronization behavior. YAML itself never enters either backend.

Recap
  • SDL3 unifies the window, event, input, and pixel-size layer; Direct3D 11 and Metal remain separate backends.
  • The shared renderer initializes from SDL_Window* plus typed VSync settings and exposes one frame lifecycle.
  • Windows obtains a borrowed HWND from SDL window properties, then owns its DXGI swap chain with ComPtr.
  • macOS owns an SDL_MetalView and borrows its layer; it never retains or releases that layer.
  • Pixel-size events call Resize, using physical dimensions on both normal and Retina displays.
  • Destroy the renderer—and therefore the Metal view or swap chain—before the SDL window.
