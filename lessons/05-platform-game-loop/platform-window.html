Creating a Window with SDL3

Our engine needs one portable platform layer and two native graphics backends. SDL3 will create the window, translate events, read input, and report drawable pixels on Windows and macOS. Direct3D 11 and Metal still render the frames, so we simplify platform code without hiding the graphics APIs we want to learn.

What you'll learn
  • Where SDL3 belongs in a cross-platform engine
  • How to fetch and link a pinned SDL3 release from CMakeLists.txt
  • How to create and destroy a high-DPI SDL window
  • How the same window reaches Direct3D 11 on Windows and Metal on macOS

What SDL3 does—and does not do

Windows normally uses Win32 for windows and messages, while macOS uses Cocoa. SDL3 wraps those operating-system differences. It does not automatically translate Direct3D 11 commands into Metal commands. We will keep a small renderer interface with a native implementation for each platform.

WindowSDL_WindowSDL exposes its HWNDSDL creates a Metal view
Events and inputSDL_PollEventSame engine code
Drawable sizeSDL_GetWindowSizeInPixelsPhysical pixels, including Retina/high-DPI
GPU workRenderer interfaceDirect3D 11Metal
Why not SDL_GPU?

SDL_GPU is SDL3's separate cross-platform graphics API. It is a useful option for a production engine, but it would replace much of the raw Direct3D/Metal work taught in Module 6. This course uses SDL3 for the platform layer and keeps native renderer backends so you learn both the abstraction and the APIs underneath it.

Step 1 — add SDL3 in CMakeLists.txt

Keep dependencies in CMake rather than an IDE-specific panel. This configuration downloads the exact SDL 3.4.12 source release during the first CMake configure, builds it with the project, and reuses it afterward.

cmake_minimum_required(VERSION 3.24)
project(PlatformWindow LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

include(FetchContent)

# A static SDL build avoids copying SDL3.dll beside the executable on Windows.
set(SDL_SHARED OFF CACHE BOOL "" FORCE)
set(SDL_STATIC ON CACHE BOOL "" FORCE)
set(SDL_TESTS OFF CACHE BOOL "" FORCE)
set(SDL_EXAMPLES OFF CACHE BOOL "" FORCE)

FetchContent_Declare(
    SDL3
    GIT_REPOSITORY https://github.com/libsdl-org/SDL.git
    GIT_TAG release-3.4.12
    GIT_SHALLOW TRUE
)
FetchContent_MakeAvailable(SDL3)

add_executable(PlatformWindow main.cpp)
target_link_libraries(PlatformWindow PRIVATE SDL3::SDL3)

In CLion, choose Tools → CMake → Reload CMake Project after saving. The first configure needs internet access and takes longer because it downloads and builds SDL3. Later configures use CMake's populated dependency directory.

The target name stays stable

Link SDL3::SDL3, not a path to a library file. That official CMake target supplies SDL's include folders, compile settings, and required platform libraries on both course platforms.

Step 2 — create the window

SDL_main.h adapts the normal C++ main entry point to each operating system. Include it exactly once, in the source file that defines main.

#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>
#include <iostream>

int main(int argc, char* argv[])
{
    if (!SDL_Init(SDL_INIT_VIDEO))
    {
        std::cerr << "SDL initialization failed: "
                  << SDL_GetError() << '\n';
        return 1;
    }

    SDL_WindowFlags flags =
        SDL_WINDOW_RESIZABLE | SDL_WINDOW_HIGH_PIXEL_DENSITY;

#if defined(__APPLE__)
    // Required before SDL_Metal_CreateView in the Metal lesson.
    flags |= SDL_WINDOW_METAL;
#endif

    SDL_Window* window = SDL_CreateWindow(
        "MiniEngine", 1280, 720, flags);

    if (window == nullptr)
    {
        std::cerr << "Window creation failed: "
                  << SDL_GetError() << '\n';
        SDL_Quit();
        return 1;
    }

    bool running = true;
    while (running)
    {
        SDL_Event event;
        while (SDL_PollEvent(&event))
        {
            if (event.type == SDL_EVENT_QUIT ||
                event.type == SDL_EVENT_WINDOW_CLOSE_REQUESTED)
            {
                running = false;
            }
        }
    }

    SDL_DestroyWindow(window);
    SDL_Quit();
    return 0;
}

Run PlatformWindow from CLion. A resizable 1280 × 720 window should open on Windows or macOS, and its close button should end the loop with exit code 0.

Logical size vs drawable pixels

A window's logical size is useful for layout; a render target needs physical pixels. A 1280 × 720 Retina window can contain roughly 2560 × 1440 drawable pixels. Query the pixel size whenever the renderer initializes or handles a pixel-size event:

int pixelWidth = 0;
int pixelHeight = 0;
if (!SDL_GetWindowSizeInPixels(window, &pixelWidth, &pixelHeight))
{
    std::cerr << SDL_GetError() << '\n';
}

// Use these dimensions for the D3D back buffer or Metal drawable.
A minimized window can have zero drawable pixels

Pause rendering and target recreation while either dimension is zero. Wait for a later SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED event instead of asking a graphics API for a zero-sized texture.

Handing the surface to a renderer

Most engine code treats SDL_Window* as an opaque platform handle. Only renderer initialization crosses the native boundary:

PlatformSDL3 bridgeRenderer use
WindowsSDL_GetWindowProperties plus SDL_PROP_WINDOW_WIN32_HWND_POINTERPass the HWND to the DXGI swap chain
macOSSDL_Metal_CreateView and SDL_Metal_GetLayerConfigure the returned CAMetalLayer

That boundary is deliberately narrow. Gameplay, scene, entity, input-action, and timing code never includes Win32, Cocoa, Direct3D, or Metal types.

Recap
  • SDL3 owns portable window, event, input, timing, and high-DPI work.
  • CMake fetches a pinned SDL3 release and links the portable SDL3::SDL3 target.
  • SDL_WINDOW_METAL is enabled only on macOS; Direct3D obtains SDL's Win32 handle on Windows.
  • Render-target dimensions come from SDL_GetWindowSizeInPixels.
  • The engine still has separate Direct3D 11 and Metal renderer implementations behind one interface.
