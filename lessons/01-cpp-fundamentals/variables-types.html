Variables & Fundamental Types

Every engine is, at heart, a giant pile of numbers changing over time: positions, colors, velocities, timestamps. Before we can move a cube or tint a pixel, we need a place to store those numbers. That place is a variable, and its type tells the compiler exactly what kind of value it holds and how much memory it takes. In this lesson you'll learn to declare, initialize, and reason about C++'s fundamental types — the raw material every component in our engine is built from.

What you'll learn
  • How to declare and initialize a variable, and why initialization matters
  • The fundamental types int, float, double, bool, and char
  • The difference between brace {} and copy = initialization
  • How to let the compiler deduce a type with auto
  • Typical type sizes, sizeof, and why graphics code lives on float

Declaring a variable

A declaration introduces a name and gives it a type. The pattern is always type, then name:

int health;        // a whole number, currently uninitialized
float speed;       // a single-precision decimal, uninitialized
bool isAlive;      // true or false, uninitialized

Those three variables exist, but they hold garbage — whatever bits happened to be sitting in that memory. Reading an uninitialized variable is one of the most common beginner bugs, and it produces values that seem to change randomly between runs.

Always initialize

An uninitialized local variable has an indeterminate value. Don't rely on it being zero — give every variable a starting value as you declare it. Your future self, debugging a jittering character at 2 a.m., will thank you.

Initialization: brace {} vs copy =

To give a variable a starting value, you initialize it. C++ offers a few syntaxes; the two you'll use most are copy initialization with = and brace (list) initialization with {}:

int   health = 100;    // copy initialization
float speed  = 5.5f;   // the trailing 'f' makes this a float literal
bool  isAlive{true};   // brace initialization
char  grade{'A'};      // a single character in single quotes

They look interchangeable, and for these examples they behave the same. But brace initialization has one very useful safety feature: it refuses to silently throw away data. This is called a narrowing conversion, and braces turn it into a compile error instead of a quiet bug.

int a = 3.9;    // compiles: quietly truncates to 3 (the .9 vanishes)
int b{3.9};     // ERROR: brace init won't narrow a double to an int
Prefer braces

Modern C++ style leans toward {} initialization precisely because it catches accidental narrowing. When we build our Transform component later, using braces means the compiler flags a bad assignment before it ever ships.

The fundamental types

C++ has a small set of built-in ("fundamental") types. These five cover almost everything you'll do early on:

TypeHoldsExample literalTypical size
intWhole numbers (positive or negative)42, -74 bytes
floatSingle-precision decimals5.5f, -0.5f4 bytes
doubleDouble-precision decimals3.141598 bytes
boolTruth valuestrue, false1 byte
charA single character / small integer'A', '\n'1 byte

A few things worth calling out:

  • float vs double: both store decimals, but double uses twice the memory for roughly twice the precision. A plain literal like 3.14 is a double; write 3.14f to make it a float.
  • char is a number too: internally 'A' is just the integer 65. You can do arithmetic on it, which is occasionally handy.
  • bool only ever holds true or false — perfect for flags like "is this entity active?"

Why graphics code lives on float

Here's a habit to lock in early: almost all graphics math uses float, not double. A GPU is built to chew through millions of 32-bit floats per frame. Vertex positions, colors, matrices, and shader variables are overwhelmingly float. Direct3D's own math library, DirectXMath, is entirely float-based.

Get comfortable with the f

When you write a graphics constant, add the f: 0.0f, 1.0f, 0.5f. It keeps the value a float and avoids needless double-to-float conversions. You'll type it thousands of times building the engine, so it becomes muscle memory fast.

Signed vs unsigned

By default int is signed — it can be negative. There's also unsigned int, which can only be zero or positive and, for the same 4 bytes, reaches about twice as high. You'll meet unsigned mostly as sizes and counts; for example, the number of elements in a std::vector is an unsigned type.

int          temperature = -15;   // can be negative
unsigned int vertexCount = 3000;  // never negative, so unsigned fits
Unsigned wraps around

Subtracting past zero in an unsigned type doesn't give a negative — it wraps to a huge number. If vertexCount is 0 and you compute vertexCount - 1, you get roughly four billion, not -1. Keep that in mind when looping backward over counts.

Letting the compiler decide with auto

Sometimes the type is obvious from the value on the right-hand side, and spelling it out is just noise. The auto keyword tells the compiler to deduce the type for you:

auto count = 10;       // deduced as int
auto ratio = 0.5f;     // deduced as float (because of the 'f')
auto pi    = 3.14159;  // deduced as double
auto ready = true;     // deduced as bool

auto shines later when types get long — think iterators or template types — where writing the full name is tedious and error-prone. For simple literals it's a matter of taste, but it's worth recognizing now because you'll see it everywhere in real engine code.

Measuring types with sizeof

The sizeof operator reports how many bytes a type occupies. Knowing this matters in graphics: when we later hand a buffer of vertices to the GPU, we tell Direct3D the exact byte size, and getting it wrong corrupts the whole mesh.

Let's put everything together in a small, complete program you can compile and run:

#include <iostream>

int main()
{
    // Declare and initialize one of each fundamental type.
    int    health = 100;      // whole number
    float  speed  = 5.5f;     // single-precision decimal (note the 'f')
    double gravity = -9.81;   // double-precision decimal
    bool   isAlive = true;    // truth value
    char   grade   = 'A';     // a single character

    // Print the values. std::boolalpha shows true/false as words.
    std::cout << "health : " << health  << '\n';
    std::cout << "speed  : " << speed   << '\n';
    std::cout << "gravity: " << gravity << '\n';
    std::cout << "isAlive: " << std::boolalpha << isAlive << '\n';
    std::cout << "grade  : " << grade   << '\n';

    // Report how many bytes each type uses on this platform.
    std::cout << "\nSizes in bytes:\n";
    std::cout << "int    = " << sizeof(int)    << '\n';
    std::cout << "float  = " << sizeof(float)  << '\n';
    std::cout << "double = " << sizeof(double) << '\n';
    std::cout << "bool   = " << sizeof(bool)   << '\n';
    std::cout << "char   = " << sizeof(char)   << '\n';

    return 0;
}

On a typical 64-bit Windows setup, this prints something like:

health : 100
speed  : 5.5
gravity: -9.81
isAlive: true
grade  : A

Sizes in bytes:
int    = 4
float  = 4
double = 8
bool   = 1
char   = 1

Notice float and int are both 4 bytes, while double is 8. That size difference is exactly why the GPU prefers float: half the memory and half the bandwidth per number, and for on-screen visuals the extra precision of double is rarely worth the cost.

Try it yourself

Change a value, swap a = for {}, or add unsigned int to the size list and rerun. Poking at working code and watching what changes is the fastest way to make these types feel natural.

These humble variables are the atoms of everything ahead. A position is three floats, a color is four, a rotation is a handful more — and a component like Transform is really just a tidy bundle of the types you met today. Master how they're declared and sized now, and the engine work later becomes a matter of arranging familiar pieces.

Recap
  • A variable declaration is type then name; always initialize it to avoid indeterminate "garbage" values.
  • The core fundamental types are int, float, double, bool, and char.
  • Brace {} initialization is safer than copy = because it rejects narrowing conversions like 3.9 into an int.
  • auto lets the compiler deduce a type from the initializer — handy for long or obvious types.
  • sizeof reports bytes; float is 4 bytes and dominates graphics code, so write literals like 1.0f.
  • unsigned types can't go negative and wrap around at zero — great for counts, but watch backward loops.
