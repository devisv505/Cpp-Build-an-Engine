Loops: while, do-while & for

In the last lesson you taught your program to make decisions. Now you'll teach it to repeat work without copying and pasting the same lines a hundred times. Loops are the beating heart of every program you'll ever write — and quite literally the heart of a game engine, which runs one enormous loop over and over to draw each frame. Let's learn the three loop shapes C++ gives us and when to reach for each.

What you'll learn
  • Write while and do-while loops and understand the difference
  • Read and write the classic for loop with its three parts
  • Steer a loop with break and continue
  • Combine loops by nesting them
  • See how a game's "main loop" runs forever, frame after frame

Why loops exist

Imagine you want to print the numbers 1 through 5. You could write five std::cout lines by hand. But what about 1 through 500? Copy-paste stops being funny very quickly, and if the rule changes you'd have to edit every line. A loop lets you describe the repetition once and let the computer do the counting. Every loop needs three ingredients: a starting point, a condition that decides whether to keep going, and something that changes each time so the loop eventually stops.

The while loop

A while loop is the simplest shape: "while this condition is true, keep running the body." It checks the condition before each pass, so if the condition is false to begin with, the body never runs at all.

#include <iostream>

int main()
{
    int count = 1;              // 1. starting point

    while (count <= 5)          // 2. condition: keep going while true
    {
        std::cout << count << " ";
        count = count + 1;      // 3. change, or the loop never ends!
    }

    std::cout << "\nDone!\n";
    return 0;
}
// Output: 1 2 3 4 5
//         Done!

Trace it in your head: count starts at 1, the condition count <= 5 is true, so we print and bump count to 2. This repeats until count becomes 6, the condition is false, and the loop exits.

The infinite loop trap

If you forget the line that changes count, the condition stays true forever and your program hangs. If a program ever freezes and spins your fan up, a runaway loop is the usual culprit. Always make sure something inside the body moves you toward the condition becoming false.

The do-while loop

Sometimes you want the body to run at least once before checking the condition — for example, showing a menu and then asking "again?" A do-while loop puts the check at the bottom.

#include <iostream>

int main()
{
    int choice = 0;

    do
    {
        std::cout << "1) Play  2) Quit\n> ";
        std::cin >> choice;          // body runs before the check
    }
    while (choice != 2);            // repeat until the user picks Quit

    std::cout << "Goodbye!\n";
    return 0;
}

Notice the semicolon after while (...) — it's required here and easy to forget. The menu always appears once, which is exactly what you want.

The for loop

When you're counting a known number of times, the for loop is the tidiest choice because it gathers all three loop ingredients onto one line. Its anatomy is for (init; condition; update).

#include <iostream>

int main()
{
    int sum = 0;

    // init: i = 1  |  condition: i <= 100  |  update: i++
    for (int i = 1; i <= 100; ++i)
    {
        sum += i;               // add each number to the running total
    }

    std::cout << "The sum of 1..100 is " << sum << "\n";
    return 0;
}
// Output: The sum of 1..100 is 5050

Read that for header left to right: create i starting at 1; before each pass check i <= 100; after each pass run ++i (add one to i). The variable i only exists inside the loop, which keeps your code clean. Any for loop can be rewritten as a while loop and vice versa — for just keeps the moving parts in one place.

Prefer ++i over i++

For plain counters the two behave the same in a loop header, but ++i ("pre-increment") is the idiomatic C++ habit. It matters later when i is a heavier type like an iterator, where i++ makes a needless copy. Start the good habit now.

Steering a loop: break and continue

Two keywords let you override a loop's normal flow. break exits the loop immediately. continue skips the rest of the current pass and jumps straight to the next one.

#include <iostream>

int main()
{
    for (int i = 1; i <= 10; ++i)
    {
        if (i == 8)
            break;              // stop the loop entirely at 8

        if (i % 2 != 0)
            continue;           // skip odd numbers, go to next i

        std::cout << i << " ";   // only even numbers below 8 reach here
    }
    std::cout << "\n";
    return 0;
}
// Output: 2 4 6

Here continue filters out odd values and break halts the loop once i hits 8, so we print 2, 4 and 6. Used sparingly, these keywords make loops read clearly; overused, they can make flow hard to follow.

Nested loops

A loop can live inside another loop. The inner loop runs completely for every single pass of the outer loop. This is how you walk a 2D grid — rows and columns — which we'll do constantly when addressing pixels or tiles.

#include <iostream>

int main()
{
    // Outer loop = rows, inner loop = columns
    for (int row = 0; row < 3; ++row)
    {
        for (int col = 0; col < 3; ++col)
        {
            std::cout << "(" << row << "," << col << ") ";
        }
        std::cout << "\n";       // new line after each row
    }
    return 0;
}
// Output:
// (0,0) (0,1) (0,2)
// (1,0) (1,1) (1,2)
// (2,0) (2,1) (2,2)
Watch your counts

Three rows times three columns means the inner body runs 9 times. Nesting multiplies work, so a loop inside a loop inside a loop over big ranges can get expensive fast. Keep an eye on how many total passes you're really asking for.

The one loop that never stops: the game loop

Here's the exciting part. A game — and the engine we're building — is fundamentally one loop that runs forever until the player quits. Each pass through that loop is a single frame: read input, update the world, draw everything to the screen, repeat. At 60 frames per second that loop runs 60 times every second.

#include <iostream>

// A rough sketch of the heartbeat we'll build for real later.
int main()
{
    bool running = true;

    while (running)                 // the main loop: one pass = one frame
    {
        // 1. Input   — has the player pressed a key or closed the window?
        // 2. Update  — move objects, run physics, tick timers
        // 3. Render  — draw the frame with DirectX

        // For now, a fake exit so this demo actually finishes:
        static int frame = 0;
        std::cout << "Rendering frame " << frame << "\n";
        if (++frame >= 3)
            running = false;        // real games flip this when you hit Quit
    }

    std::cout << "Window closed. Bye!\n";
    return 0;
}

Notice it's just a while loop with a bool condition — nothing you haven't already met. When we reach the DirectX modules, those three commented steps become real function calls, and the loop keeps your triangle (and eventually your whole 3D scene) alive on screen. Every loop you practise now is direct preparation for that engine heartbeat.

C++ also has a range-based for loop for stepping through every item in a collection without a counter. It's a perfect fit for arrays and std::vector, so we'll save it for the arrays lesson where it truly shines. Just know it exists.

Recap
  • Every loop needs a start, a condition, and something that changes each pass — or it never ends.
  • while checks first; do-while runs the body once then checks; for packs all three parts into its header.
  • Use a for loop when you're counting a known number of times, like summing 1 through 100.
  • break leaves the loop; continue skips to the next pass.
  • Nesting loops walks 2D grids, but multiplies the total number of passes.
  • A game (and our engine) is one big loop running frame after frame until the player quits.
