Operators & Expressions

Now that you can print to the console and store values in constants, it is time to make the computer actually compute. Operators are the verbs of C++: they add, compare, and combine values to form expressions. Every line of engine math — moving a Transform, checking whether an object is on screen, ticking a timer — is built from the small handful of operators you will meet here.

What you'll learn
  • The arithmetic operators + - * / % and the famous integer-division gotcha
  • Comparison operators that produce true or false
  • Logical operators &&, || and ! for combining conditions
  • Assignment, compound assignment, and the increment/decrement operators
  • How operator precedence decides what runs first — and how to control it

Expressions and operators

An expression is any piece of code that produces a value. 3 is an expression, and so is width * height. An operator is a symbol that combines one or more values (its operands) into a new value. Operators are the fundamental building blocks you will use constantly when we start computing positions, velocities and colors for the engine.

Arithmetic operators

C++ gives you the five arithmetic operators you already know from school, plus one that is new to many beginners:

OperatorMeaningExampleResult
+Addition7 + 29
-Subtraction7 - 25
*Multiplication7 * 214
/Division7 / 23 (see below!)
%Modulo (remainder)7 % 21

The % operator (called modulo) gives you the remainder after division. 7 % 2 is 1 because 2 goes into 7 three times with 1 left over. Modulo only works on integers, and it is surprisingly useful: we will use it later to wrap animation frames around or to tell whether a number is even (n % 2 == 0).

#include <iostream>

int main() {
    int apples = 17;
    int baskets = 5;

    int perBasket = apples / baskets;   // 3  (integer division drops the remainder)
    int leftover  = apples % baskets;   // 2  (17 = 5*3 + 2)

    std::cout << perBasket << " apples per basket, "
              << leftover  << " left over\n";
    return 0;
}

The integer-vs-float division gotcha

Here is the single most common surprise for new C++ programmers. When both operands are integers, / performs integer division: it throws away the fractional part instead of rounding. So 7 / 2 is 3, not 3.5. The decimal simply vanishes.

Integer division truncates — and modulo needs integers

If either operand is a floating-point value, the division becomes real division. 7 / 2 is 3, but 7.0 / 2 is 3.5. To force a float result from two int variables, cast one first: static_cast<float>(a) / b. And remember % is integer-only — 7.5 % 2 will not compile; use std::fmod for floating-point remainders.

#include <iostream>

int main() {
    int a = 7, b = 2;

    std::cout << (a / b) << "\n";                       // 3   -- both ints, truncated
    std::cout << (7.0 / 2) << "\n";                     // 3.5 -- one operand is a double
    std::cout << (static_cast<float>(a) / b) << "\n";   // 3.5 -- cast forces float division

    // This matters for real engine math, e.g. an aspect ratio:
    int width = 1280, height = 720;
    float aspect = static_cast<float>(width) / height; // 1.777...  (NOT 1)
    std::cout << "aspect ratio: " << aspect << "\n";
    return 0;
}

That aspect-ratio example is not hypothetical — when we set up the DirectX projection matrix, computing it as plain width / height would give 1 and stretch everything into a square. This tiny gotcha is a real bug you will otherwise chase for an hour.

Comparison operators

Comparison operators ask a yes/no question about two values and produce a bool — either true or false. These feed directly into the decisions you will learn next lesson.

OperatorMeaninga = 5, b = 8
==Equal toa == b → false
!=Not equal toa != b → true
<Less thana < b → true
>Greater thana > b → false
<=Less than or equala <= 5 → true
>=Greater than or equalb >= 8 → true
One equals sign is not two

= assigns a value; == compares. Writing if (x = 5) silently stores 5 into x and is almost always a bug. Modern compilers warn about it — do not ignore the warning.

Logical operators

Real conditions are often combinations: "the player is alive and has ammo," or "the key is Escape or the window closed." Logical operators join bool values together.

  • && (AND) — true only when both sides are true.
  • || (OR) — true when at least one side is true.
  • ! (NOT) — flips a bool: !true is false.
#include <iostream>

int main() {
    int health = 80;
    int ammo   = 0;
    bool isVisible = true;

    bool canFight = (health > 0) && (ammo > 0);   // false: alive but out of ammo
    bool shouldDraw = isVisible && (health > 0);  // true

    std::cout << std::boolalpha;   // print true/false instead of 1/0
    std::cout << "canFight:   " << canFight   << "\n";
    std::cout << "shouldDraw: " << shouldDraw << "\n";
    std::cout << "is dead?    " << !(health > 0) << "\n"; // false
    return 0;
}
Short-circuit evaluation

&& and || are lazy. If the left side of && is already false, C++ never evaluates the right side (the answer is settled). This lets you safely write guards like if (ptr != nullptr && ptr->alive) — the pointer check protects the access that follows.

Increment and decrement

Adding or subtracting exactly one is so common that C++ has dedicated operators: ++ and --. Each has a prefix form (++x) and a postfix form (x++). Both change x the same way; they differ only in the value the expression itself hands back. Prefix increments first and returns the new value; postfix returns the old value, then increments.

#include <iostream>

int main() {
    int frame = 0;

    frame += 1;   // frame is now 1  (compound assignment)
    ++frame;      // frame is now 2  (prefix increment)
    frame++;      // frame is now 3  (postfix increment)

    int a = 5;
    int b = a++;  // b gets 5 (old value), THEN a becomes 6
    int c = ++a;  // a becomes 7 first, THEN c gets 7

    std::cout << "frame=" << frame << " a=" << a
              << " b=" << b << " c=" << c << "\n"; // frame=3 a=7 b=5 c=7
    return 0;
}
Prefix vs postfix in practice

When you use ++x as a standalone statement — which is most of the time, including loop counters — the two forms behave identically, so pick whichever reads well. The difference only matters when you use the result in a larger expression. When we write our game loop, you will see ++frameCount; ticking over every frame.

Operator precedence

When an expression mixes operators, C++ needs rules for what runs first — exactly like "multiply before add" in ordinary math. This ordering is called precedence. Here is a simplified table, highest priority at the top:

PriorityOperatorsGroup
Highest()Parentheses (grouping)
!   ++   --Unary
*   /   %Multiplicative
+   -Additive
<   <=   >   >=Relational
==   !=Equality
&&Logical AND
||Logical OR
Lowest=   +=   -=   …Assignment

So 2 + 3 * 4 is 14, not 20, because * outranks +. And because comparison outranks &&, the expression health > 0 && ammo > 0 is read as (health > 0) && (ammo > 0) — which is exactly what you want.

When in doubt, parenthesize

You never have to memorize the whole table. Parentheses have the highest precedence and make your intent obvious to both the compiler and the next human reading the code. (a + b) * c is clearer than trusting everyone to recall the rules.

#include <iostream>

int main() {
    int result = 2 + 3 * 4;        // 14  -- * before +
    int forced = (2 + 3) * 4;      // 20  -- parentheses win

    // A realistic mix: clamp-style check on a score threshold
    int score = 42;
    bool passed = score >= 40 && score % 2 == 0; // (score>=40) && ((score%2)==0)

    std::cout << result << " " << forced << " "
              << std::boolalpha << passed << "\n"; // 14 20 true
    return 0;
}
Recap
  • Arithmetic operators are + - * / %; % gives the integer remainder.
  • Integer / truncates — cast to float (or use a floating operand) when you need a decimal result, as with an aspect ratio.
  • Comparison operators produce a bool; remember == compares while = assigns.
  • Logical &&, || and ! combine conditions and short-circuit from left to right.
  • Compound assignment (+=, *=, …) and ++/-- are concise shorthands.
  • Precedence decides evaluation order; parentheses always let you take control.
