Functions: Reusing Your Code

So far your programs have run top to bottom as one long list of statements. Real software — and especially a 3D engine — is built from small, named pieces of behaviour you can call over and over. Those pieces are functions. In this lesson you'll learn to package logic into functions, pass data in, get results back, and give several functions the same name. Every engine system we build later (a Transform, a renderer, a math library) is really just a well-organised pile of functions.

What you'll learn
  • How to declare, define and call a function with parameters and a return type
  • What void means and when a function returns nothing
  • Why parameters are passed by value (as copies) by default
  • How function overloading and default arguments reduce repetitive code
  • The difference between a declaration (prototype) and a definition, and why order matters

Anatomy of a function

A function has four parts: a return type, a name, a list of parameters in parentheses, and a body in braces. Here is a tiny function that squares a number: 

// return type   name   parameter list
int Square(int value)
{
    return value * value;   // hand a result back to the caller
}

The int before the name says "this function evaluates to an int". The return statement produces that value and immediately ends the function. To use the function you call it by writing its name and supplying arguments: 

#include <iostream>

int Square(int value)
{
    return value * value;
}

int main()
{
    int result = Square(5);           // result is now 25
    std::cout << "5 squared is " << result << "\n";
    std::cout << "9 squared is " << Square(9) << "\n"; // call it inline too
    return 0;
}
Parameter vs. argument

The parameter is the placeholder in the function's definition (value). The argument is the actual data you pass at the call site (5). People use the words loosely, but knowing the distinction helps when reading compiler errors.

Parameters and return types

A function can take zero, one, or many parameters, each with its own type, separated by commas. It returns exactly one value whose type matches the return type. Here is a function that picks the larger of two integers: 

int Max(int a, int b)
{
    if (a > b)
        return a;   // one exit...
    return b;       // ...or the other
}

You can have multiple return statements; whichever one runs first ends the function. The value you return must be convertible to the declared return type — returning a double from an int function would silently chop off the fractional part, which is rarely what you want.

Functions that return nothing: void

Sometimes a function does something rather than computing something — it prints, it updates state, it draws. Such a function has the return type void, meaning "no value comes back". You may still use a bare return; to exit early. 

#include <iostream>
#include <string>

void PrintBanner(const std::string& title)
{
    if (title.empty())
        return;   // nothing to print, leave early

    std::cout << "==== " << title << " ====\n";
}
You've already seen void and int

Your programs start in int main(). That's just a function the operating system calls for you; its returned int is the program's exit code (0 means success).

Pass-by-value: functions get copies

By default, a parameter receives a copy of the argument. Whatever the function does to that copy does not touch the caller's original variable. This is called pass-by-value. 

#include <iostream>

void TryToDouble(int n)
{
    n = n * 2;   // only changes this function's local copy
    std::cout << "inside: " << n << "\n";
}

int main()
{
    int score = 10;
    TryToDouble(score);
    std::cout << "outside: " << score << "\n"; // still 10!
    return 0;
}

This prints:

inside: 20
outside: 10

The copy makes functions safe and predictable — a function can't accidentally scramble your data. The trade-off is cost: copying a small int is free, but copying a big object (say a mesh with thousands of vertices) every call would be wasteful.

Coming next module: pass-by-reference

When you want a function to modify the caller's variable, or you want to avoid an expensive copy, you'll pass by reference (int&) or by const reference. That's exactly how our engine's systems will update a Transform in place. We introduce it properly in the next module — for now, just know copies are the default.

Overloading: one name, several versions

C++ lets you give several functions the same name as long as their parameter lists differ in number or type. The compiler picks the right one based on the arguments you pass. This is called function overloading, and it keeps related operations under one friendly name. 

int Square(int value)
{
    return value * value;
}

// Same name, different parameter type — a distinct function.
double Square(double value)
{
    return value * value;
}

// Overloads can also differ in how MANY parameters they take.
int Max(int a, int b)          { return a > b ? a : b; }
int Max(int a, int b, int c)   { return Max(Max(a, b), c); }

Now Square(5) calls the int version and Square(2.5) calls the double version. Note that the return type alone cannot distinguish overloads — the difference has to be in the parameters.

CallWhich overload runsResult
Square(6)Square(int)36
Square(1.5)Square(double)2.25
Max(3, 9)Max(int, int)9
Max(3, 9, 4)Max(int, int, int)9

Default arguments

You can give a parameter a default value, used when the caller omits that argument. This is handy for "usually this, occasionally that" options. Defaults must come last in the parameter list — once one parameter has a default, every parameter after it needs one too. 

#include <iostream>

// step defaults to 1 if the caller doesn't supply it.
void CountUp(int to, int step = 1)
{
    for (int i = 0; i <= to; i += step)
        std::cout << i << ' ';
    std::cout << "\n";
}

int main()
{
    CountUp(5);       // uses step = 1  -> 0 1 2 3 4 5
    CountUp(10, 2);   // step = 2       -> 0 2 4 6 8 10
    return 0;
}
Overloads or defaults?

Default arguments and overloading solve overlapping problems. Reach for a default when the versions share one body and differ only by a value; reach for overloads when the versions genuinely do different work or take different types.

Declaration vs. definition — why order matters

C++ reads your file from top to bottom. Before you can call a function, the compiler must already know its name, parameters and return type. If you define a function below main and call it from inside main, the compiler hasn't met it yet and errors out.

The fix is a declaration (also called a prototype): the function's signature followed by a semicolon, placed near the top. It's a promise — "this function exists; here's its shape — the body appears later." The full function, braces and all, is the definition. 

#include <iostream>

// --- Declarations (prototypes): promises to the compiler ---
int Square(int value);
int Max(int a, int b);

int main()
{
    // These calls compile because the prototypes are already visible.
    std::cout << Square(7) << "\n";   // 49
    std::cout << Max(3, 8) << "\n";    // 8
    return 0;
}

// --- Definitions: the actual bodies, order no longer matters ---
int Square(int value)
{
    return value * value;
}

int Max(int a, int b)
{
    return a > b ? a : b;
}
This is how header files work

When our engine grows, declarations move into .h header files and definitions into .cpp files. Including a header gives every file the prototypes it needs, while each function is defined in exactly one place. You'll do this for real when we split code across files.

Always return a value from non-void functions

If a function promises to return an int but a code path reaches the end without a return, the returned value is undefined — your program may print garbage or crash unpredictably. Make sure every path returns. Turn on compiler warnings and the compiler will flag it for you.

Putting it together

Here's a small, complete program that uses everything above: prototypes at the top, an overloaded Square, a default argument, and clear void output. Compile and run it to see the pieces cooperate. 

#include <iostream>
#include <string>

// Prototypes
int    Square(int value);
double Square(double value);
int    Max(int a, int b);
void   Report(const std::string& label, int value, int width = 8);

int main()
{
    int   a = 6;
    double b = 2.5;

    Report("Square(int)",    Square(a));   // 36
    Report("Square(double)", static_cast<int>(Square(b))); // 6 (2.5*2.5 = 6.25 -> truncated)
    Report("Max",            Max(a, 9), 12);
    return 0;
}

// Definitions
int Square(int value)          { return value * value; }
double Square(double value)    { return value * value; }
int Max(int a, int b)          { return a > b ? a : b; }

void Report(const std::string& label, int value, int width)
{
    std::cout << label;
    // pad with spaces so the numbers line up
    for (int i = static_cast<int>(label.size()); i < width; ++i)
        std::cout << ' ';
    std::cout << "= " << value << "\n";
}

Functions like Square and Max are exactly the kind of small, reusable helpers that fill an engine's math library. Once you can factor logic into clean, well-named functions, the rest of the course is mostly about writing more of them — and organising them into components.

Recap
  • A function bundles reusable logic behind a name; you call it with arguments and it may return a value.
  • The return type declares what comes back; void means nothing does, and a bare return; exits early.
  • Parameters are passed by value (copies) by default, so changes inside a function don't affect the caller — pass-by-reference comes next module.
  • Overloading lets several functions share a name if their parameter lists differ; default arguments fill in omitted values.
  • A declaration/prototype tells the compiler a function's shape so you can call it before its definition appears — the basis of header files.
