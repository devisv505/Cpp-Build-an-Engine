Procedural Terrain with Noise

Your engine can render a chunk of blocks, but right now every one is placed by hand. A real voxel world is built by a function: hand it a chunk coordinate and it fills the chunk with hills, valleys, beaches, and caves — the same landscape every time, from a single integer seed. The tool that makes this possible is coherent noise. In this lesson we turn a seed into terrain: a deterministic GenerateChunk that neighbouring chunks can run independently and still stitch together seam-free.

Random is not the same as natural

The obvious first idea is to give every voxel a random height or a random "solid/air" flag. It fails immediately. A call like rand() produces white noise: each value is completely independent of its neighbours, so adjacent columns jump from height 5 to 90 to 12. You get static, not a landscape. Worse, it is not repeatable — walk away and come back and the ground is different.

What terrain needs is coherent noise, which has three properties white noise lacks:

  • Smooth — nearby inputs give nearby outputs, so the surface flows instead of jittering.
  • Continuous — defined at every real-numbered point, not just on a grid, so you can sample it at any resolution.
  • Repeatable — the same input always returns the same output. This is what lets separately generated chunks agree on their shared border.

Three families of coherent noise show up in voxel engines. It is worth knowing them by name even though you will not hand-write them:

NoiseIdeaFeel & cost
ValueRandom value at each lattice corner, interpolated betweenSimplest to understand; can look slightly blocky/grid-aligned
Gradient (Perlin)Random gradient at each corner; interpolate the dot productsSmoother, no visible grid; the classic terrain choice
Simplex / OpenSimplexPerlin's successor on a triangular latticeFewer directional artifacts and much cheaper in 3D+

Hand-rolled Perlin is a rite of passage, but for the engine reach for a proven single header: stb_perlin.h (Perlin noise with ready-made fBm, turbulence, and ridged helpers) or FastNoiseLite (which adds Simplex/OpenSimplex, built-in fBm, and domain warping). Both are seedable, fast, and correct across negative coordinates. We build the mechanics below so it is not a black box — but we ship the library.

What "coherent" looks like under the hood

Here is the smallest coherent noise there is — value noise in two dimensions — so the idea is concrete. Imagine an integer grid. At each corner we hash a pseudo-random value in [0, 1). To sample an arbitrary point, we find the four surrounding corners and blend their values, using a smooth fade curve on the blend factor so the result has no creases where cells meet.

#include <cstdint>
#include <cmath>

// A deterministic hash of integer lattice coordinates -> [0, 1).
// Same (x, z, seed) always yields the same value: that is coherence.
static float Hash01(int x, int z, uint32_t seed) {
    uint32_t h = seed;
    h ^= uint32_t(x) * 374761393u;
    h ^= uint32_t(z) * 668265263u;
    h  = (h ^ (h >> 13)) * 1274126177u;
    h ^= h >> 16;
    return float(h) / float(0xFFFFFFFFu);
}

// Smootherstep: zero 1st and 2nd derivatives at 0 and 1, so cell
// boundaries do not crease. This is the "fade" curve Perlin uses.
static float Fade(float t) { return t * t * t * (t * (t * 6.0f - 15.0f) + 10.0f); }
static float Lerp(float a, float b, float t) { return a + (b - a) * t; }

// 2D value noise. Continuous, smooth, repeatable. Returns ~[0, 1].
float ValueNoise2D(float x, float z, uint32_t seed) {
    float fx = std::floor(x), fz = std::floor(z);
    int   x0 = int(fx),       z0 = int(fz);
    float tx = Fade(x - fx),  tz = Fade(z - fz);

    float c00 = Hash01(x0,     z0,     seed);   // four surrounding
    float c10 = Hash01(x0 + 1, z0,     seed);   // lattice corners
    float c01 = Hash01(x0,     z0 + 1, seed);
    float c11 = Hash01(x0 + 1, z0 + 1, seed);

    return Lerp(Lerp(c00, c10, tx),             // bilinear blend,
                Lerp(c01, c11, tx), tz);        // smoothed by Fade
}

That is the whole trick: a hash pins down repeatable randomness at grid corners, and a smooth interpolation fills the space between. Perlin noise replaces the corner values with random gradients and interpolates their dot products, which removes the faint grid you can sometimes see in value noise; Simplex does the same on a triangular lattice that scales far better into 3D. All three return a single smooth number for any point. From here on we treat noise as one such function — noise2D(x, z) and noise3D(x, y, z), each returning roughly [-1, 1] — and let FastNoiseLite provide it.

Stacking octaves: fractal Brownian motion

A single octave of noise gives smooth, rolling hills — but real terrain has detail at every scale: mountain ranges, individual hills, and boulder-sized bumps all at once. We get that by summing several copies of the same noise at different scales. Each successive copy (an octave) is sampled at a higher frequency and added with a smaller amplitude. This sum is called fractal Brownian motion, or fBm.

The loop is short. Two knobs drive it: lacunarity multiplies the frequency each octave (≈2, so detail gets twice as fine), and gain (a.k.a. persistence) multiplies the amplitude each octave (≈0.5, so each layer is half as loud). We track the total amplitude so we can normalize the sum back to [-1, 1].

#pragma once

// Fractal Brownian motion: stack octaves of a coherent noise field.
// 'noise' is any callable returning ~[-1, 1] for a single octave.
template <class NoiseFn>
float fbm2D(NoiseFn noise, float x, float z,
            int octaves, float frequency,
            float lacunarity, float gain) {
    float sum = 0.0f, amplitude = 1.0f, ampSum = 0.0f;

    for (int o = 0; o < octaves; ++o) {
        sum       += amplitude * noise(x * frequency, z * frequency);
        ampSum    += amplitude;      // remember total for normalization
        frequency *= lacunarity;     // each octave: finer detail...
        amplitude *= gain;           // ...contributing less
    }
    return sum / ampSum;             // normalized back to ~[-1, 1]
}

Dividing by ampSum matters: without it, adding octaves would push the range past 1 and the terrain scale would drift as you changed the octave count. With it, octaves controls only how much fine detail appears, not the overall height. Here is the full parameter set and what each one does:

ParameterTypicalEffect
seedany intSelects which world you get; identical seed → identical terrain
frequency0.005–0.02Base horizontal scale. Lower = wider, gentler features
amplitude16–64 blocksHow tall hills and mountains rise (applied after fBm)
octaves3–6Layers of detail. More = rougher; diminishing returns past ~6
lacunarity2.0Frequency multiplier per octave
gain0.5Amplitude multiplier per octave (persistence)

From heightmap to voxel columns

Now the payoff. For every world column (x, z) we compute one surface height, then fill the voxels beneath it. fBm returns roughly [-1, 1], so we center it on a base height and scale by an amplitude:

float e      = fbm2D(noise2D, wx, wz, 4, 0.008f, 2.0f, 0.5f); // -1..1
int   height = baseHeight + int(e * amplitude);  // e.g. 64 + e*24

Determinism and world coordinates: the seam bug

Every chunk is generated on its own, often on a different thread, with no knowledge of its neighbours. The only thing that keeps their shared edges continuous is that both chunks sample the same noise function at the same world position and get the same answer. That gives us a hard rule.

Never feed local coordinates to noise

If you sample noise2D(lx, lz) with the chunk-local 0..CHUNK-1 coordinates, every chunk generates the same hill and the world tiles like wallpaper — and the heights jump discontinuously at every border. This visible cliff at chunk boundaries is the single most common terrain bug. Always convert to world coordinates first: wx = chunkOrigin.x + lx, then sample noise2D(wx, wz).

Determinism buys you more than seamless borders. Because GenerateChunk is a pure function of (seed, chunkCoord) — no globals, no clock, no rand() state — the same chunk regenerates identically forever. That is what lets you (a) generate chunks on worker threads safely, which we do in the job-system lesson, and (b) skip saving untouched chunks entirely: you can rebuild them from the seed instead of writing them to disk.

Keep generation pure

Seed every noise field once from the world seed and read only from the chunk coordinate. No shared mutable state, no time, no I/O. A pure GenerateChunk is automatically thread-safe, reproducible for saves, and trivial to unit-test: assert that two calls with the same input produce byte-identical chunks.

Biomes: a second, slower noise picks the palette

A world of identical grass hills is dull. Biomes add variety by layering a second noise field, sampled at a much lower frequency so regions are large and change gradually. Classic Minecraft uses two — temperature and humidity — and looks up a biome from the pair. The biome then changes both the terrain shape (amplitude, octaves) and the block palette.

#pragma once
#include "BlockRegistry.h"   // Blocks::Air, Stone, Dirt, Grass, Sand, Water...

struct Biome {
    int   baseHeight;        // sea-ish level the terrain centers on
    int   amplitude;         // how tall its hills climb
    int   octaves;
    float frequency, lacunarity, gain;
    BlockId surface, subsurface, filler;  // grass/dirt/stone, sand/sand/stone...
};

// A small hand-authored set. Real engines interpolate between many more.
inline const Biome kPlains   { 64, 14, 4, 0.008f, 2.0f, 0.5f, Blocks::Grass, Blocks::Dirt, Blocks::Stone };
inline const Biome kDesert   { 66,  8, 3, 0.006f, 2.0f, 0.5f, Blocks::Sand,  Blocks::Sand, Blocks::Stone };
inline const Biome kMountains{ 80, 52, 5, 0.010f, 2.0f, 0.5f, Blocks::Grass, Blocks::Dirt, Blocks::Stone };
Blend, don't switch, for smooth borders

Picking a biome with a hard if makes a mountain slam straight into a desert at a one-block cliff. Production worlds sample the neighbourhood and interpolate the numeric parameters (amplitude, base height) between biomes, so shape transitions are gradual even where the palette flips. We keep the hard switch here for clarity; treat blending as the next refinement.

Caves and overhangs: 3D density noise

A heightmap can only describe one surface per column — it can never carve a cave or hang an overhang, because "solid below the height, air above" has no room for a hole in the middle. For that we need genuine 3D noise. We sample noise3D(x, y, z) and treat it as a density field: wherever the density crosses a threshold, we scoop the voxel out to air, even deep inside the rock.

// After a voxel has been classified as solid rock, maybe carve it.
// noise3D returns ~[-1, 1]; a high threshold carves a small fraction
// of space, giving sparse, connected pockets rather than swiss cheese.
float d = noise3D(wx * CAVE_FREQ, wy * CAVE_FREQ, wz * CAVE_FREQ);
if (d > CAVE_THRESHOLD)          // e.g. CAVE_FREQ = 0.04, THRESHOLD = 0.6
    block = Blocks::Air;         // hollowed out -> a cave or an overhang

Raise the threshold for rarer, smaller caves; lower it and you flood the world with holes. Because this is a 3D field, the same mechanism naturally produces overhangs and floating arches near the surface — geometry a pure heightmap simply cannot express. A popular alternative for long, winding tunnels is worm caves: trace a random walk (a "worm") through the world and carve a tube of blocks around its path. Density noise gives blobby caverns; worms give corridors; most engines use both.

Putting it together: GenerateChunk

Now assemble everything into the one function the rest of the engine calls. We assume the earlier lessons' cubic Chunk (edge CHUNK) with a SetBlock(lx, ly, lz, id), a 3D ChunkCoord, and the block ids from the registry. The generator owns four independently seeded noise fields so terrain, caves, temperature, and humidity never accidentally correlate. Each is set to SetFrequency(1.0f) so our multipliers fully control scale.

#pragma once
#include "FastNoiseLite.h"
#include "fbm.h"
#include "Biomes.h"
#include "Chunk.h"

constexpr int   CHUNK          = 32;      // cubic chunk edge (from lesson 1)
constexpr int   SEA_LEVEL      = 62;
constexpr int   BEDROCK        = 0;       // lowest solid world layer
constexpr float BIOME_FREQ     = 0.0008f; // very low -> large biomes
constexpr float CAVE_FREQ      = 0.04f;
constexpr float CAVE_THRESHOLD = 0.6f;

struct ChunkCoord { int x, y, z; };

class TerrainGenerator {
public:
    explicit TerrainGenerator(int seed) {
        // Four fields, four seeds. Same world seed -> same world.
        Configure(m_height, seed);
        Configure(m_cave,   seed + 1);
        Configure(m_temp,   seed + 2);
        Configure(m_humid,  seed + 3);
    }

    void GenerateChunk(ChunkCoord coord, Chunk& chunk) const;

private:
    static void Configure(FastNoiseLite& n, int seed) {
        n.SetSeed(seed);
        n.SetNoiseType(FastNoiseLite::NoiseType_OpenSimplex2);
        n.SetFrequency(1.0f);   // we scale coordinates ourselves
    }

    // fBm surface height, delegating single-octave sampling to m_height.
    float Height(int wx, int wz, const Biome& b) const {
        auto sample = [this](float x, float z) { return m_height.GetNoise(x, z); };
        return fbm2D(sample, float(wx), float(wz),
                     b.octaves, b.frequency, b.lacunarity, b.gain);
    }

    const Biome& SelectBiome(int wx, int wz) const {
        float t = m_temp .GetNoise(wx * BIOME_FREQ, wz * BIOME_FREQ); // -1..1
        float h = m_humid.GetNoise(wx * BIOME_FREQ, wz * BIOME_FREQ);
        if (t > 0.4f && h < 0.0f) return kDesert;
        if (t < -0.2f)            return kMountains;
        return kPlains;
    }

    FastNoiseLite m_height, m_cave, m_temp, m_humid;
};
#include "TerrainGenerator.h"

void TerrainGenerator::GenerateChunk(ChunkCoord coord, Chunk& chunk) const {
    const int ox = coord.x * CHUNK;   // this chunk's world origin
    const int oy = coord.y * CHUNK;
    const int oz = coord.z * CHUNK;

    for (int lx = 0; lx < CHUNK; ++lx)
    for (int lz = 0; lz < CHUNK; ++lz) {
        const int wx = ox + lx;       // WORLD coords -- never local!
        const int wz = oz + lz;

        // One height + one biome per column (independent of y).
        const Biome& biome = SelectBiome(wx, wz);
        const int height = biome.baseHeight
                         + int(Height(wx, wz, biome) * biome.amplitude);

        for (int ly = 0; ly < CHUNK; ++ly) {
            const int wy = oy + ly;
            BlockId block = Blocks::Air;

            if (wy < height) {
                // Layered palette from the surface down.
                if (wy == height - 1)
                    block = (height <= SEA_LEVEL + 1) ? Blocks::Sand   // beach
                                                      : biome.surface;  // grass
                else if (wy >= height - 4)
                    block = biome.subsurface;                           // dirt
                else
                    block = biome.filler;                               // stone
            } else if (wy < SEA_LEVEL) {
                block = Blocks::Water;    // air below the sea fills with water
            }

            // Carve caves & overhangs from solid rock only.
            if (block == biome.filler || block == biome.subsurface) {
                float d = m_cave.GetNoise(wx * CAVE_FREQ,
                                          wy * CAVE_FREQ,
                                          wz * CAVE_FREQ);
                if (d > CAVE_THRESHOLD && wy > BEDROCK)
                    block = Blocks::Air;
            }

            chunk.SetBlock(lx, ly, lz, block);
        }
    }
}

Read the flow once more: convert to world space first, take one height and biome per column, fill the column with the layered palette, flood the low air with water, then carve caves from the rock. Nothing here touches a clock, a global counter, or a neighbouring chunk — so the same (seed, coord) always yields the same blocks, and two chunks that share an edge compute identical heights along it.

Note that terrain generation is entirely CPU-side and renderer-agnostic: this exact code runs identically on Windows and macOS. Only afterward does the chunk go through the meshing and upload path from the earlier module lessons — Direct3D 11 buffers on Windows, Metal buffers on macOS — with no platform-specific logic in the generator itself.

Watch float precision far from the origin

Because we key everything to world coordinates, a player who walks millions of blocks out will be sampling noise at huge x/z values, where 32-bit float steps between adjacent integers grow and terrain can get subtly warped. It rarely bites within a normal play area, but if you plan for effectively infinite worlds, sample noise in double or periodically re-origin the world.

Recap
  • Per-voxel rand() is white noise — useless for terrain. You need coherent noise: smooth, continuous, and repeatable.
  • Value, Perlin (gradient), and Simplex are the three families; understand the mechanics but ship a proven single header like FastNoiseLite or stb_perlin.h.
  • fBm sums octaves at rising frequency (lacunarity ≈ 2) and falling amplitude (gain ≈ 0.5), normalized by total amplitude, to get multi-scale terrain.
  • A heightmap fills each column — stone, then dirt, a surface cap, water below sea level — while a low-frequency biome field varies amplitude and palette, and 3D density noise carves caves and overhangs.
  • Always sample noise at world coordinates (chunkOrigin + local); local coordinates tile the world and crack every chunk border — the #1 terrain bug.
  • Keep GenerateChunk a pure function of (seed, coord): it stays thread-safe for the job system, reproducible for saving, and seam-free by construction.
