Your First Program: Hello, World

Now that the tools are installed, it's time to make the compiler do something for you. In this lesson you'll create a CMake executable in CLion, look at the classic "Hello, World" program, and take it apart one token at a time. The same project works on Windows and macOS.

What you'll learn
  • How to create a new C++ executable project in CLion
  • What every token in the Hello World program means
  • How #include, int main(), and return 0 fit together
  • How std::cout and the << operator print text
  • How to build and run your program and see its output

Creating a CMake executable

Open CLion and choose New Project. Select C++ Executable, which creates a small CMake project that CLion can build on either operating system.

  1. Give the project a name, for example HelloWorld.
  2. Pick a location on disk you can find again — you'll be creating many projects over this course.
  3. Select C++20 as the language standard and click Create.

CLion generates main.cpp and CMakeLists.txt. CMake describes the executable target; main.cpp contains its code. Wait for the initial CMake load to finish, then replace main.cpp with a clean version.

cmake_minimum_required(VERSION 3.20)
project(HelloWorld LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(HelloWorld main.cpp)
A .cpp file is just text

The .cpp extension marks a C++ source file. It's plain text — the compiler reads it and translates it into an executable your computer can run. Nothing magic about it.

The program

Select everything in main.cpp, delete it, and type this in. Typing it yourself (rather than pasting) helps it stick:

#include <iostream>

int main()
{
    std::cout << "Hello, World\n";
    return 0;
}

Six meaningful lines. Let's walk through every one.

#include <iostream>

Lines starting with # are preprocessor directives — instructions handled before the real compilation begins. #include means "paste the contents of another file right here." The file <iostream> is part of the C++ Standard Library; its name stands for input/output stream. It declares the tools we need to print text to the screen, including std::cout. Without this line, the compiler would have no idea what std::cout is.

The angle brackets < and > tell the compiler to look for the file among the standard, system-provided headers. Later we'll write our own headers and include them with quotes instead, like #include "Transform.h".

int main()

This line declares a function named main. A function is a named block of instructions. main is special: it is the entry point of every C++ program. When you run your program, the operating system calls main first, and execution flows from top to bottom through its body.

The word int in front says main hands back an integer (a whole number) when it finishes. The empty parentheses () mean it takes no inputs for now. The curly braces { } that follow enclose the function's body — the actual work.

Exactly one main

A program has exactly one main function — it's the one, agreed-upon place execution starts. Every game engine, including the one you'll build, ultimately has a main somewhere that kicks everything off.

std::cout << "Hello, World\n";

This is the line that actually does something visible. It has several parts:

  • std::cout is the standard output stream — a channel that sends text to the console window. The std:: prefix means it lives in the standard namespace, a labelled container that groups all the Standard Library's names so they don't collide with yours. Read it aloud as "std, then cout."
  • << is the stream insertion operator. It "pushes" the thing on its right into the stream on its left. Think of the arrows as pointing the data toward cout.
  • "Hello, World\n" is a string literal — literal text, marked by double quotes. Everything between the quotes is printed exactly as written, except for escape sequences (next paragraph).
  • The ; at the end is a semicolon. In C++ almost every statement ends with one. Forgetting it is the single most common beginner error, so watch for it.

The \n inside the quotes is an escape sequence meaning "newline." The backslash tells the compiler the next character is special rather than literal, and n makes the cursor drop to the next line — just like pressing Enter. Without it, later output would run on the same line.

Chain your output

You can insert several things in one statement by chaining <<: std::cout << "Score: " << 42 << "\n";. Each << hands another value to the stream, left to right. We'll lean on this constantly when logging what our engine is doing.

return 0;

Remember that main promised to return an int. This line fulfils that promise. Returning 0 is the universal convention for "the program finished successfully." Any non-zero value signals that something went wrong, which is useful when other programs or scripts run yours and check whether it succeeded. After return, the function ends and the program exits.

Build and run

Time to see it work. Select the HelloWorld CMake run configuration in the top toolbar. CLion creates it automatically from add_executable.

  • Click the green Run triangle beside main or in the toolbar. CLion configures CMake, builds the target, and launches it.
  • The Run tool window shows:
Hello, World

Process finished with exit code 0

That exit-code line is your return 0; showing up in the real world: the operating system received the success code. The output looks the same on Windows and macOS.

If it won't compile

Read the first error in CLion's Build tool window, not the last — later errors are often fallout from the first. Common causes are a missing ;, a mismatched quote or brace, or a typo like Std::cout. Fix the first error and build again.

Congratulations — you just wrote, compiled, and ran a C++ program, and you can explain every character in it. That's a real milestone. In the next lesson we'll dig deeper into the build-and-run cycle and meet the debugger, the single most valuable tool for understanding what your code is actually doing — knowledge you'll rely on heavily once your engine grows to thousands of lines.

Recap
  • A CLion C++ executable project uses CMakeLists.txt to turn main.cpp into a runnable target.
  • #include <iostream> pulls in the Standard Library's input/output tools, including std::cout.
  • int main() is the program's entry point; execution runs top to bottom through its { } body.
  • std::cout << "text\n"; sends text to the console, and \n starts a new line; every statement ends in ;.
  • return 0; ends main and reports success to the operating system.
  • Use the green Run triangle to build and launch the selected CMake target.
