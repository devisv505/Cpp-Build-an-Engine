What Is C++ & Why Build Engines With It

C++ is the language that powers Unreal Engine, most of Unity's core, and virtually every high-performance game engine on the planet. Before we write a single line, let's understand what makes it the right tool for the job you're about to take on: building your own real-time DirectX engine from scratch.

What exactly is C++?

C++ is a compiled, statically-typed systems programming language. That's a mouthful, so let's unpack it:

  • Compiled — your source code is translated ahead of time into raw machine instructions the CPU runs directly. There's no interpreter or virtual machine sitting in the middle slowing things down.
  • Statically typed — the type of every variable is known at compile time, so the compiler can catch mistakes and generate extremely efficient code.
  • Systems language — it gives you unrestricted access to memory, hardware, and operating-system APIs like DirectX.

C++ started life in 1985 as "C with classes" — an extension of the C language that added object-oriented features. Today it's a mature, standardized language (we'll target C++17/20) that still keeps C's philosophy: you don't pay for what you don't use, and you stay close to the metal.

C vs C++

C++ is (almost) a superset of C. Everything you learn about variables, functions, and pointers is shared. C++ simply adds classes, templates, and a large standard library on top. Game engines lean heavily on those additions.

Why do game engines use C++?

A real-time engine has to update physics, animation, audio, and render millions of triangles — all within roughly 16 milliseconds to hit 60 frames per second. That budget is brutal, and it's exactly where C++ shines.

1. Predictable, high performance

C++ compiles to native machine code with no garbage collector pausing your game at random moments. You decide when memory is allocated and freed, which means no surprise frame-rate hitches.

2. Direct hardware & API access

DirectX, Vulkan, and the Windows API are all exposed as C-style interfaces. C++ can call them directly, hand raw memory buffers to the GPU, and manage them with zero overhead.

A taste of the language

Here's a tiny C++ program. Don't worry about understanding every symbol yet — we'll build all of this up carefully. Just notice that it reads fairly clearly:

#include <iostream>

// Every C++ program starts running from main()
int main()
{
    const char* engineName = "MyEngine";
    int targetFps = 60;

    std::cout << "Booting " << engineName
              << " at " << targetFps << " FPS\n";

    return 0; // 0 means "success"
}

That #include <iostream> line pulls in the standard input/output library, main() is where execution begins, and std::cout prints text. We'll dissect every piece of this in the coming lessons.
