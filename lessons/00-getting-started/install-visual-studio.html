Installing Visual Studio, CLion & the Tools

In the last lesson you learned what a compiler and linker do. Now it's time to put them on your machine. On Windows, we'll install Visual Studio and the Windows SDK so you're ready for C++ and DirectX 11. On macOS, we'll install Apple Clang and CLion so you can follow the C++ lessons with the same professional editing, building, and debugging tools.

What you'll learn

  • How to install Apple Clang and CLion on a Mac

Windows — Visual Studio

Visual Studio vs. Visual Studio Code

First, a name that trips up almost everyone. Visual Studio and Visual Studio Code are two different products. Visual Studio Code (VS Code) is a lightweight text editor. Visual Studio is the full development environment (an "IDE") that bundles the Microsoft C++ compiler, a debugger, and the Windows SDK together. For this course we want the full Visual Studio, because it ships everything DirectX needs in one place.

It really is free

The Community edition is completely free for students, open-source projects, and individual developers. There is no trial timer and no credit card required — you will build this entire engine without paying a cent.

Step 1 — Download the installer

Open your browser and go to the Visual Studio downloads page. On Windows 11, choose Visual Studio Community 2026. This downloads a small installer — it's only a few megabytes because it's just the bootstrapper. The real components download in the next step once you've told it what you want.

Still using Windows 10?

Visual Studio 2026 requires Windows 11. Install Visual Studio Community 2022 from Microsoft's older downloads instead; the workload choices and course instructions are the same. Windows 10 has reached end of support, so upgrade to Windows 11 when your hardware allows it.

Step 2 — Run the installer and pick the workload

Double-click the downloaded file. After a short setup step, the Visual Studio Installer window appears with a grid of workloads. A workload is a bundle of tools for a particular kind of development. You only need one:

  • Find the tile labeled Desktop development with C++ and tick its checkbox. This installs the MSVC compiler, the linker, the standard library, and the debugger you read about last lesson.

Before clicking Install, look at the Installation details panel on the right. This lists the individual components the workload will add. This is the part that matters most for us.

Confirm a Windows SDK is checked

In that right-hand panel, make sure a Windows 11 SDK (or Windows 10 SDK) checkbox is ticked. The Windows SDK contains the DirectX 11 headers and libraries — files like d3d11.h and d3d11.lib — that our engine will include and link against. The "Desktop development with C++" workload usually selects a recent SDK for you, but always verify it, because without it none of the DirectX code in later lessons will compile.

Don't skip the SDK

If you later see errors like cannot open source file "d3d11.h", it almost always means the Windows SDK component wasn't installed. You can fix it any time by reopening the Visual Studio Installer and clicking Modify.

The table below shows what to expect. The first two rows are the essentials; the SDK is the one to double-check.

Step 3 — Verify the install

Once the installer finishes, let's prove the compiler is really there. The simplest check is to launch Visual Studio from the Start menu — if it opens to the start window, the IDE is installed. But the more satisfying check is to ask the compiler itself to report its version.

Visual Studio installs a special terminal named Developer PowerShell for VS, followed by your Visual Studio version. It's a normal PowerShell window that already knows where the compiler lives. Open the Start menu, type "Developer PowerShell", and launch it. Then run:

cl

cl is the name of the Microsoft C++ compiler (short for "compiler"). With no arguments it prints its version and exits. You should see something similar to this:

Microsoft (R) C/C++ Optimizing Compiler Version 19.5x for x64
Copyright (C) Microsoft Corporation.  All rights reserved.

usage: cl [ option... ] filename... [ /link linkoption... ]

If you see that, congratulations — your toolchain works. If instead PowerShell says cl is not recognized, you almost certainly opened a normal PowerShell rather than the Developer one; the developer version sets up the environment variables the compiler needs.

Prefer to confirm the DirectX headers too?

You can check the SDK arrived by asking whether the main DirectX header exists. The exact folder version will differ on your machine, and that's fine.

# List any d3d11.h the Windows SDK installed (version folders vary)
Get-ChildItem "C:\Program Files (x86)\Windows Kits\10\Include" -Recurse -Filter d3d11.h

If that command prints one or more paths, the DirectX 11 headers are on your system and you're ready for the engine chapters.

A tiny end-to-end test (optional)

Want to compile something right now? In the Developer PowerShell, create a one-line program and build it. This exercises the compiler and the linker exactly as we discussed in the previous lesson.

#include <iostream>

int main()
{
    // If this prints, your compiler and linker both work.
    std::cout << "Toolchain OK\n";
    return 0;
}
# Compile check.cpp into check.exe, then run it
cl /EHsc check.cpp
.\check.exe

Seeing Toolchain OK in the terminal means the whole pipeline — source file, compiler, linker, executable — is healthy. The /EHsc flag simply enables standard C++ exception handling; we'll rely on it throughout the course.

Don't run cl from a plain terminal

Calling cl from an ordinary Command Prompt or PowerShell will fail with "not recognized." The compiler needs environment variables that only the Developer terminals set up. Always use the Developer PowerShell for your Visual Studio version (or build from inside Visual Studio) and this problem disappears.

macOS — CLion and Apple Clang

CLion is a cross-platform C++ IDE from JetBrains. It provides the editor, project system, and debugger, while Apple's developer tools provide the actual Clang compiler and macOS SDK. CLion also bundles CMake, so you do not need to install CMake separately for this course.

CLion is free for learning

JetBrains offers CLion free for non-commercial use, including learning, hobby projects, and non-commercial open-source work. Choose Non-commercial use when CLion asks for a license and sign in with a JetBrains Account. Commercial work requires a paid license.

Step 1 — Install Apple's command-line tools

Open Terminal from Applications → Utilities and run:

xcode-select --install

macOS opens an installation dialog. Click Install, accept the license, and wait for it to finish. This adds Apple Clang, the macOS SDK, system headers, and other tools that CLion needs. If the command says the tools are already installed, you can continue. A full Xcode installation also includes these tools, so you do not need to install them twice.

Step 2 — Verify Apple Clang

Still in Terminal, ask the C++ compiler for its version:

clang++ --version

You should see output beginning with Apple clang version. The exact version and target depend on your version of macOS and whether your Mac uses Apple silicon or Intel.

After a macOS upgrade

If Clang suddenly stops working after upgrading macOS, open System Settings → General → Software Update and install any available Command Line Tools update. Apple sometimes releases a matching tools package separately.

Step 3 — Install CLion

Go to the CLion download page. JetBrains recommends installing it through the Toolbox App, which handles updates for you. A standalone download also works: open the .dmg and drag CLion into your Applications folder.

Launch CLion, choose Non-commercial use for this course, and sign in or create a JetBrains Account. When the welcome screen appears, the IDE itself is installed.

Step 4 — Check CLion's toolchain

In CLion, open Settings with ⌘,, then go to Build, Execution, Deployment → Toolchains. CLion normally detects everything automatically. Your default system toolchain should show:

ToolExpected choice
C compilerApple Clang (/usr/bin/clang)
C++ compilerApple Clang++ (/usr/bin/clang++)
DebuggerBundled LLDB
CMakeBundled
Build toolBundled Ninja or detected Make

Green check marks beside the entries mean CLion found the tools. If the compiler fields are empty, make sure clang++ --version works in Terminal, restart CLion, and return to this screen.

Step 5 — Compile the test program on macOS

Save the same check.cpp program shown above, open Terminal in that folder, and run:

# Compile with C++17, create an executable named check, then run it
clang++ -std=c++17 check.cpp -o check
./check

If Terminal prints Toolchain OK, Apple Clang compiled and linked your program successfully. CLion will use that same compiler when you build projects inside the IDE.

Optional — Use CLion on Windows

Prefer CLion's interface on Windows? You can use it without giving up DirectX support. Install Visual Studio and the Desktop development with C++ workload first, exactly as described in the Windows path above. Then install CLion and select a Visual Studio toolchain under Settings → Build, Execution, Deployment → Toolchains.

Keep the Microsoft toolchain on Windows

CLion is the IDE, but MSVC and the Windows SDK still do the compiling and provide the DirectX headers and libraries. Avoid switching the course projects to MinGW if you want the smoothest path through the Windows and DirectX chapters.

Possible errors and how to fix them

Toolchain errors can look like problems in your C++ even when the code is correct. Match the message you see below, apply the fix, and then let the IDE reload its project information.

macOS — fatal error: 'iostream' file not found

This means Clang is installed, but it cannot find the C++ standard-library headers. It commonly happens when the active Command Line Tools package is incomplete or no longer matches macOS after an update.

First, check which developer tools macOS is using:

xcode-select -p
xcrun --find clang++
xcrun --show-sdk-path

If full Xcode is installed in the Applications folder, switch to its complete toolchain:

sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer

Enter your Mac password when Terminal asks for it, then verify that the selected path and compiler changed:

xcode-select -p
xcrun clang++ --version

The first command should now print /Applications/Xcode.app/Contents/Developer. Restart CLion and select Tools → CMake → Reset Cache and Reload Project before building again.

Don't have full Xcode?

Open System Settings → General → Software Update and install any available Command Line Tools update. If no compatible update appears and xcode-select --install does not repair the missing headers, install Xcode from the Mac App Store and use the switch command above.

CLion — the project builds, but standard-library names stay red

If the program runs with exit code 0, but <iostream> and std::cout are red or typing std:: shows no suggestions, the compiler is working but CLion's editor index still has the old header paths cached.

  1. Select Tools → CMake → Reset Cache and Reload Project.
  2. Wait until CMake loading and indexing finish; progress appears in CLion's status bar.
  3. If names remain red, open Settings → Build, Execution, Deployment → Toolchains. Confirm the C++ compiler is /usr/bin/clang++ and the tool entries have green check marks, then click Apply.
  4. Reset the CMake cache once more. As a final fallback, select File → Invalidate Caches → Invalidate and Restart.
Build output and editor analysis are separate

A successful build proves that the compiler can find the headers. Red code after that usually means the IDE's cached code model is stale, not that you need to change #include <iostream> or add a special library.

Why this setup matters for your engine

It might feel like a lot of ceremony just to print a line of text, but you now have the tools that turn source code into a running program. On Windows, MSVC compiles your C++ and the Windows SDK provides DirectX files such as d3d11.h and d3d11.lib. On macOS, Apple Clang compiles the same standard C++ while CLion uses CMake to organize the build and LLDB to debug it. The core language skills transfer between both platforms; only the platform-specific graphics layer changes.

Recap
  • On Windows 11, install Visual Studio Community 2026 with the Desktop development with C++ workload; Windows 10 users can use Visual Studio 2022.
  • Confirm a Windows 10/11 SDK is checked so the DirectX 11 headers and libraries are available.
  • Verify MSVC by running cl in the Developer PowerShell for your Visual Studio version.
  • On macOS, install Apple's tools with xcode-select --install, then install CLion.
  • Verify Apple Clang with clang++ --version and check CLion's system toolchain.
  • Mac users can complete the C++ lessons natively, but the DirectX engine chapters require Windows.
