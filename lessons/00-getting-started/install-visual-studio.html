Installing Visual Studio & the Tools

In the last lesson you learned what a compiler and linker do. Now it's time to actually put them on your machine. We'll install Visual Studio 2022 Community — Microsoft's free, professional-grade toolset — and configure it precisely so it can compile C++ and talk to DirectX 11. Get this right once and every later lesson, all the way to your own 3D engine, will just work.

What you'll learn
  • Where to download Visual Studio 2022 Community and why it's free for you
  • How to run the installer and pick the right workload
  • Which components matter for DirectX — especially the Windows 10/11 SDK
  • How to confirm the compiler is installed and working from a terminal

Visual Studio vs. Visual Studio Code

First, a name that trips up almost everyone. Visual Studio and Visual Studio Code are two different products. Visual Studio Code (VS Code) is a lightweight text editor. Visual Studio 2022 is the full development environment (an "IDE") that bundles the Microsoft C++ compiler, a debugger, and the Windows SDK together. For this course we want the full Visual Studio, because it ships everything DirectX needs in one place.

It really is free

The Community edition is completely free for students, open-source projects, and individual developers. There is no trial timer and no credit card required — you will build this entire engine without paying a cent.

Step 1 — Download the installer

Open your browser and go to visualstudio.microsoft.com. Under the "Download Visual Studio" button, choose the Community 2022 edition. This downloads a small file named something like VisualStudioSetup.exe — it's only a few megabytes because it's just the bootstrapper. The real components download in the next step once you've told it what you want.

Windows only, for now

DirectX 11 is a Windows technology, so this course assumes Windows 10 or 11. If you're on macOS or Linux you can still learn the C++ half, but the DirectX engine chapters need Windows (a virtual machine or dual-boot works fine).

Step 2 — Run the installer and pick the workload

Double-click the downloaded file. After a short setup step, the Visual Studio Installer window appears with a grid of workloads. A workload is a bundle of tools for a particular kind of development. You only need one:

  • Find the tile labeled Desktop development with C++ and tick its checkbox. This installs the MSVC compiler, the linker, the standard library, and the debugger you read about last lesson.

Before clicking Install, look at the Installation details panel on the right. This lists the individual components the workload will add. This is the part that matters most for us.

Confirm a Windows SDK is checked

In that right-hand panel, make sure a Windows 11 SDK (or Windows 10 SDK) checkbox is ticked. The Windows SDK contains the DirectX 11 headers and libraries — files like d3d11.h and d3d11.lib — that our engine will include and link against. The "Desktop development with C++" workload usually selects a recent SDK for you, but always verify it, because without it none of the DirectX code in later lessons will compile.

Don't skip the SDK

If you later see errors like cannot open source file "d3d11.h", it almost always means the Windows SDK component wasn't installed. You can fix it any time by reopening the Visual Studio Installer and clicking Modify.

When the boxes are ticked, click Install. The download is several gigabytes, so this is a good moment to grab a coffee. You can keep working on the machine while it runs.

Step 3 — Verify the install

Once the installer finishes, let's prove the compiler is really there. The simplest check is to launch Visual Studio 2022 from the Start menu — if it opens to the start window, the IDE is installed. But the more satisfying check is to ask the compiler itself to report its version.

Visual Studio installs a special terminal called the Developer PowerShell for VS 2022. It's a normal PowerShell window that already knows where the compiler lives. Open the Start menu, type "Developer PowerShell", and launch it. Then run:

cl

cl is the name of the Microsoft C++ compiler (short for "compiler"). With no arguments it prints its version and exits. You should see something similar to this:

Microsoft (R) C/C++ Optimizing Compiler Version 19.4x for x64
Copyright (C) Microsoft Corporation.  All rights reserved.

usage: cl [ option... ] filename... [ /link linkoption... ]

If you see that, congratulations — your toolchain works. If instead PowerShell says cl is not recognized, you almost certainly opened a normal PowerShell rather than the Developer one; the developer version sets up the environment variables the compiler needs.

Prefer to confirm the DirectX headers too?

You can check the SDK arrived by asking whether the main DirectX header exists. The exact folder version will differ on your machine, and that's fine.

# List any d3d11.h the Windows SDK installed (version folders vary)
Get-ChildItem "C:\Program Files (x86)\Windows Kits\10\Include" -Recurse -Filter d3d11.h

If that command prints one or more paths, the DirectX 11 headers are on your system and you're ready for the engine chapters.

A tiny end-to-end test (optional)

Want to compile something right now? In the Developer PowerShell, create a one-line program and build it. This exercises the compiler and the linker exactly as we discussed in the previous lesson.

#include <iostream>

int main()
{
    // If this prints, your compiler and linker both work.
    std::cout << "Toolchain OK\n";
    return 0;
}
# Compile check.cpp into check.exe, then run it
cl /EHsc check.cpp
.\check.exe

Seeing Toolchain OK in the terminal means the whole pipeline — source file, compiler, linker, executable — is healthy. The /EHsc flag simply enables standard C++ exception handling; we'll rely on it throughout the course.

Don't run cl from a plain terminal

Calling cl from an ordinary Command Prompt or PowerShell will fail with "not recognized." The compiler needs environment variables that only the Developer terminals set up. Always use "Developer PowerShell for VS 2022" (or build from inside Visual Studio) and this problem disappears.

Why this setup matters for your engine

It might feel like a lot of ceremony just to print a line of text, but you've now assembled every ingredient our engine needs. The MSVC compiler will turn your C++ classes — Transform, MeshRenderer, and the rest — into machine code. The linker will stitch them together with the DirectX libraries. And the Windows SDK is what makes #include <d3d11.h> resolve to real, working DirectX 11 code. You won't have to think about any of this again; it just quietly powers everything ahead.

Recap
  • Install Visual Studio 2022 Community — it's free, with no trial limit.
  • In the installer, select the Desktop development with C++ workload.
  • Confirm a Windows 10/11 SDK is checked — it provides the DirectX 11 headers and libraries.
  • Verify the toolchain by running cl in the Developer PowerShell for VS 2022.
  • Optionally compile a one-line program to prove the compiler and linker work end to end.
