Text: std::string

Every engine is full of text: the name of a component, the path to a texture, the tag on a game object, lines read from a config file. In this lesson we meet std::string, the standard, safe, easy-to-use way to hold text in C++. You already know how std::vector gives you a growable list of values; a string is that same idea, specialized for characters, with a friendly toolbox bolted on.

What you'll learn
  • Create strings and join them together with +
  • Measure length, read individual characters, and slice out pieces with substr
  • Compare text with == and read whole lines with std::getline
  • Reach the underlying C-string with c_str() when an old API asks for one

Creating a string

Include <string>, then declare one just like any other variable. A string can start empty, hold a literal, or be built from another string.

#include <iostream>
#include <string>

int main() {
    std::string empty;                 // "" — zero characters
    std::string name = "Transform";    // from a text literal
    std::string copy = name;           // an independent copy

    std::cout << "Component: " << name << "\n";
    std::cout << "Empty? "     << empty.empty() << "\n";  // prints 1 (true)
    return 0;
}

Unlike a raw array of characters, a std::string owns its memory and grows on demand. You never have to guess a maximum size or remember to free anything — when the string goes out of scope, its storage is released automatically.

Joining text with +

The + operator concatenates strings, and += appends to an existing one. This is how you assemble file paths, log messages, and generated names.

#include <iostream>
#include <string>

int main() {
    std::string folder = "assets/";
    std::string file   = "player";
    std::string ext    = ".png";

    std::string path = folder + file + ext;   // "assets/player.png"
    path += "?v=2";                            // append in place

    std::cout << path << "\n";                  // assets/player.png?v=2
    return 0;
}
At least one side must be a string

You cannot write "a" + "b" — those are two C-string literals and + is not defined for them. Make one an actual string first: std::string("a") + "b", or store the first part in a std::string variable and add to that.

How long is it?

Two methods report the number of characters: size() and length(). They are identical twins — length() reads nicely for text, while size() matches the name used by std::vector and other containers.

std::string tag = "Enemy";
std::cout << tag.size()   << "\n";   // 5
std::cout << tag.length() << "\n";   // 5 — same value

Reading individual characters

A string is indexed exactly like a vector: positions start at 0, so the last valid index is size() - 1. Use square brackets to read or change a single char.

#include <iostream>
#include <string>

int main() {
    std::string word = "mesh";
    std::cout << word[0] << "\n";   // 'm'
    std::cout << word[3] << "\n";   // 'h'

    word[0] = 'M';                 // strings are mutable
    std::cout << word << "\n";      // "Mesh"

    // Walk every character
    for (char c : word) {
        std::cout << c << ' ';
    }
    std::cout << "\n";              // M e s h
    return 0;
}
Stay in range

Indexing past the end (for example word[10] on a 4-character string) is undefined behavior — it may crash or silently corrupt data. If you want a checked access that throws on a bad index, use word.at(10) instead.

Slicing with substr

substr(start, count) returns a brand-new string copied from part of the original. Give it a starting index and how many characters to take; leave off the count to grab everything to the end. This is perfect for pulling a file extension or a name out of a path.

#include <iostream>
#include <string>

int main() {
    std::string path = "assets/player.png";

    // find() returns the index of a character, or std::string::npos if absent
    std::size_t dot = path.find('.');
    if (dot != std::string::npos) {
        std::string ext = path.substr(dot + 1);   // "png" — from dot to end
        std::cout << "Extension: " << ext << "\n";
    }

    std::string first4 = path.substr(0, 4);        // "asse" — 4 chars from 0
    std::cout << first4 << "\n";
    return 0;
}

Comparing text

Here is a place where strings are much friendlier than the old C way: you compare them with ==, !=, and even < (which orders them alphabetically). The comparison checks the actual characters, which is almost always what you want.

std::string a = "Player";
std::string b = "Player";
std::string c = "Enemy";

bool same    = (a == b);   // true  — identical contents
bool differ  = (a != c);   // true
bool ordered = (c < a);    // true  — "Enemy" sorts before "Player"

Reading whole lines

Using std::cin >> text stops at the first space, so it only ever reads one word. To capture an entire line — spaces and all — use std::getline. Engines lean on this constantly when parsing config files, level descriptions, or console commands.

#include <iostream>
#include <string>

int main() {
    std::cout << "Name your entity: ";

    std::string line;
    std::getline(std::cin, line);   // reads the full line into 'line'

    std::cout << "Created entity \"" << line << "\" "
              << "(" << line.size() << " chars)\n";
    return 0;
}

C-strings and c_str()

Before std::string, C++ inherited C's way of storing text: a raw char array ending in a hidden '\0' (the null terminator), passed around as a const char*. These C-strings have no built-in length and no +; you manipulate them with functions like strlen, and it is easy to overflow a buffer. This is exactly the manual, error-prone bookkeeping std::string saves you from.

You still meet C-strings at API boundaries, including GLFW and native Windows/macOS APIs. When a function wants a const char*, hand it c_str(), which returns a read-only, null-terminated view.

#include <cstdio>
#include <string>

int main() {
    std::string shaderPath = "shaders/basic"; // backend adds .hlsl or .metal

    // std::fopen is an old C function; it needs a const char*, not a std::string
    std::FILE* f = std::fopen(shaderPath.c_str(), "rb");
    if (f) {
        std::fclose(f);
    }
    return 0;
}
Prefer std::string in your own code

Keep text as a std::string everywhere you can and only call c_str() at the very moment a C-style API demands it. The pointer c_str() returns is valid only while the string still exists and stays unchanged — never store it and use it later after modifying the string.

Putting it together

Here is a small program that uses several of these tools at once. It reads a line, tidies it up, and reports on it — the kind of text massaging you will do when loading engine assets.

#include <iostream>
#include <string>

int main() {
    std::cout << "Enter a texture path: ";
    std::string path;
    std::getline(std::cin, path);          // e.g. "textures/Brick.PNG"

    if (path.empty()) {
        std::cout << "No path given.\n";
        return 0;
    }

    // Split the folder from the file name at the last '/'
    std::size_t slash = path.find_last_of('/');
    std::string file  = (slash == std::string::npos)
                        ? path                       // no folder present
                        : path.substr(slash + 1);    // just the file name

    // Pull out the extension after the last '.'
    std::size_t dot   = file.find_last_of('.');
    std::string ext   = (dot == std::string::npos)
                        ? std::string("")
                        : file.substr(dot + 1);

    std::string summary = "File '" + file + "' has extension '" + ext + "'";
    std::cout << summary << "\n";
    std::cout << "Path length: " << path.size() << " characters\n";

    if (path == file) {
        std::cout << "(It sits in the current folder.)\n";
    }
    return 0;
}

Notice how naturally it reads: no manual memory, no length counters, no separate compare function. That readability is why our engine will store names, tags, and paths as std::string — for instance, every game object will carry a std::string name, and each component type will report its type as a string we can log and compare.

Recap
  • std::string (from <string>) is a safe, growable, self-managing container of text.
  • Build strings with + and +=, measure them with size()/length(), and read characters with [] or the checked at().
  • substr slices out copies, find locates characters, and ==/< compare contents directly.
  • std::getline reads a whole line including spaces; c_str() hands a const char* to C-style APIs such as GLFW.
  • Prefer std::string throughout your engine and drop down to a raw C-string only at the boundary that requires it.
