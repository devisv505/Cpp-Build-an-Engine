Sockets on Windows & macOS

The socket API is almost the same on both course platforms. Windows provides Winsock2; macOS provides BSD/POSIX sockets. We will isolate their small lifecycle and type differences in one header so every later networking lesson builds from the same CLion project.

What you'll learn
  • Include and link the correct networking API on Windows and macOS.
  • Normalize socket handles, closing, errors, and non-blocking mode.
  • Use RAII for Windows socket subsystem startup.
  • Create a UDP socket with one shared C++ code path.

A portable socket header

#pragma once

#ifdef _WIN32
  #define WIN32_LEAN_AND_MEAN
  #include <winsock2.h>
  #include <ws2tcpip.h>

  using SocketHandle = SOCKET;
  using SocketLength = int;
  constexpr SocketHandle InvalidSocket = INVALID_SOCKET;

  inline int CloseSocket(SocketHandle socket) { return closesocket(socket); }
  inline int LastSocketError() { return WSAGetLastError(); }
  inline bool WouldBlock(int error) { return error == WSAEWOULDBLOCK; }
  inline bool SetNonBlocking(SocketHandle socket) {
      u_long enabled = 1;
      return ioctlsocket(socket, FIONBIO, &enabled) == 0;
  }
#else
  #include <arpa/inet.h>
  #include <cerrno>
  #include <fcntl.h>
  #include <netinet/in.h>
  #include <sys/socket.h>
  #include <unistd.h>

  using SocketHandle = int;
  using SocketLength = socklen_t;
  constexpr SocketHandle InvalidSocket = -1;

  inline int CloseSocket(SocketHandle socket) { return close(socket); }
  inline int LastSocketError() { return errno; }
  inline bool WouldBlock(int error) { return error == EWOULDBLOCK || error == EAGAIN; }
  inline bool SetNonBlocking(SocketHandle socket) {
      const int flags = fcntl(socket, F_GETFL, 0);
      return flags != -1 && fcntl(socket, F_SETFL, flags | O_NONBLOCK) != -1;
  }
#endif

class SocketRuntime {
public:
    SocketRuntime() {
#ifdef _WIN32
        WSADATA data{};
        m_ready = WSAStartup(MAKEWORD(2, 2), &data) == 0;
#else
        m_ready = true;
#endif
    }

    ~SocketRuntime() {
#ifdef _WIN32
        if (m_ready) WSACleanup();
#endif
    }

    bool Ready() const { return m_ready; }

private:
    bool m_ready = false;
};

Create a socket once

#include "SocketPlatform.h"
#include <iostream>

int main() {
    SocketRuntime runtime;
    if (!runtime.Ready()) {
        std::cerr << "Socket subsystem initialization failed\n";
        return 1;
    }

    SocketHandle socket = ::socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
    if (socket == InvalidSocket) {
        std::cerr << "socket() failed: " << LastSocketError() << '\n';
        return 1;
    }

    std::cout << "UDP socket ready\n";
    CloseSocket(socket);
}

CMake in CLion

cmake_minimum_required(VERSION 3.24)
project(SocketDemo LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 20)
add_executable(SocketDemo main.cpp)

if(WIN32)
  target_link_libraries(SocketDemo PRIVATE ws2_32)
endif()

macOS needs no extra socket library. Reload CMake in CLion after editing this file, then run the same target on either platform.

Do not expose platform socket types

Engine headers should use SocketHandle and helper functions from this file. That prevents later gameplay and serialization code from becoming Windows-only.

Recap
  • Winsock2 and BSD sockets share the core calls: socket, bind, sendto, and recvfrom.
  • SocketPlatform.h hides handles, close functions, errors, and non-blocking setup.
  • SocketRuntime performs Windows-only startup with RAII and is a no-op on macOS.
  • CMake links ws2_32 only inside its WIN32 branch.
