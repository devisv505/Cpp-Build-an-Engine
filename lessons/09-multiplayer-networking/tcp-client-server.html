A Portable TCP Client & Server

We will make two programs talk over TCP. Both include the SocketPlatform.h wrapper from the previous lesson, so their networking flow is identical on Windows and macOS.

What you'll learn
  • Follow the TCP server flow: socket, bind, listen, accept, receive, send.
  • Follow the client flow: socket, address, connect, send, receive.
  • Build two CLion targets with CMake.
  • Test both processes on localhost.

The echo server

#include "SocketPlatform.h"
#include <iostream>

int main() {
    SocketRuntime runtime;
    if (!runtime.Ready()) return 1;

    SocketHandle listener = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (listener == InvalidSocket) return 1;

    sockaddr_in address{};
    address.sin_family = AF_INET;
    address.sin_addr.s_addr = htonl(INADDR_LOOPBACK);
    address.sin_port = htons(7777);

    if (bind(listener, reinterpret_cast<sockaddr*>(&address), sizeof(address)) != 0 ||
        listen(listener, 1) != 0) {
        std::cerr << "bind/listen failed: " << LastSocketError() << '\n';
        CloseSocket(listener);
        return 1;
    }

    std::cout << "Waiting on 127.0.0.1:7777...\n";
    sockaddr_in clientAddress{};
    SocketLength clientLength = sizeof(clientAddress);
    SocketHandle client = accept(listener,
        reinterpret_cast<sockaddr*>(&clientAddress), &clientLength);

    char buffer[512]{};
    const int received = recv(client, buffer, sizeof(buffer) - 1, 0);
    if (received > 0) {
        std::cout << "Client said: " << buffer << '\n';
        send(client, buffer, received, 0);
    }

    CloseSocket(client);
    CloseSocket(listener);
}

The client

#include "SocketPlatform.h"
#include <iostream>
#include <string>

int main() {
    SocketRuntime runtime;
    if (!runtime.Ready()) return 1;

    SocketHandle socketHandle = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (socketHandle == InvalidSocket) return 1;

    sockaddr_in server{};
    server.sin_family = AF_INET;
    server.sin_port = htons(7777);
    inet_pton(AF_INET, "127.0.0.1", &server.sin_addr);

    if (connect(socketHandle, reinterpret_cast<sockaddr*>(&server), sizeof(server)) != 0) {
        std::cerr << "connect failed: " << LastSocketError() << '\n';
        CloseSocket(socketHandle);
        return 1;
    }

    const std::string message = "Hello from the client";
    send(socketHandle, message.data(), static_cast<int>(message.size()), 0);

    char reply[512]{};
    const int received = recv(socketHandle, reply, sizeof(reply) - 1, 0);
    if (received > 0) std::cout << "Server replied: " << reply << '\n';

    CloseSocket(socketHandle);
}

Two targets in CLion

cmake_minimum_required(VERSION 3.24)
project(TcpDemo LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 20)

add_executable(TcpServer server.cpp)
add_executable(TcpClient client.cpp)

if(WIN32)
  target_link_libraries(TcpServer PRIVATE ws2_32)
  target_link_libraries(TcpClient PRIVATE ws2_32)
endif()
  1. Select and run TcpServer in CLion.
  2. Allow it through the Windows firewall if prompted. On macOS, allow incoming connections if the system asks.
  3. Run TcpClient in a second Run configuration.
  4. Both Run tool windows should print the same message.
TCP is a byte stream

One send is not guaranteed to match one recv. Real protocols prefix messages with a length or delimiter and loop until all requested bytes have arrived.

Recap
  • The server binds, listens, and accepts; the client connects.
  • After connection, both sides use the same send/recv stream API.
  • The portable socket wrapper makes this source work on Windows and macOS.
  • CMake creates separate server/client targets and links Winsock only on Windows.
