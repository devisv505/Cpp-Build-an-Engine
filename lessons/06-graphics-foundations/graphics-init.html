Initializing the Graphics Backend: Direct3D 11 & Metal

SDL3 gave us one window and event layer for both platforms. It does not make Direct3D calls work on macOS or Metal calls work on Windows. This lesson adds the narrow platform seam: an SDL window enters the selected renderer backend, which creates the native GPU and presentation objects.

What you'll learn
  • Which code SDL3 unifies and which code remains renderer-specific
  • How CMake selects Direct3D 11 or Metal while sharing the same application
  • How SDL exposes an HWND to DXGI on Windows
  • How SDL creates a Metal view and exposes its CAMetalLayer on macOS
  • How to initialize and shut down every resource in a safe order
Follow one backend, understand both

Windows students compile the Direct3D 11 sections; macOS students compile the Metal sections. The public class has the same name and lifecycle on both platforms, so the shared application code is identical.

The common responsibilities

Engine responsibilityDirect3D 11 on WindowsMetal on macOS
Select a GPUID3D11DeviceMTL::Device
Submit commandsID3D11DeviceContextMTL::CommandQueue
Present to SDL's windowIDXGISwapChain using an HWNDCA::MetalLayer from an SDL Metal view
Address the back bufferID3D11RenderTargetViewA new CA::MetalDrawable each frame

The responsibilities match; the object models do not. In particular, Metal has no exact equivalent of D3D11's immediate context or DXGI swap chain. Our renderer interface unifies engine operations, not every native class.

SDL3 is shared; CMake compiles exactly one native renderer. The macOS path no longer enables Objective-C++ or links Cocoa for a hand-written view bridge—SDL_Metal_CreateView performs that platform work.

Get metal-cpp on macOS

Apple's metal-cpp headers provide C++ interfaces for Foundation, Metal, and QuartzCore. From the project root, clone the tested release rather than a moving default branch:

git clone --depth 1 \
  --branch release/metal-cpp_macOS26_iOS26 \
  https://github.com/apple/metal-cpp.git external/metal-cpp

The repository is header-only, but its private implementations must be emitted in exactly one translation unit:

The shared application entry point

This is still the SDL window from Module 5. The only compile-time choice is which header defines the platform's Renderer class.

#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>
#include <iostream>

#if defined(_WIN32)
    #include "D3D11Renderer.h"
#elif defined(__APPLE__)
    #include "MetalRenderer.h"
#endif

int main(int argc, char* argv[])
{
    if (!SDL_Init(SDL_INIT_VIDEO))
    {
        std::cerr << "SDL initialization failed: "
                  << SDL_GetError() << '\n';
        return 1;
    }

    SDL_WindowFlags flags =
        SDL_WINDOW_RESIZABLE | SDL_WINDOW_HIGH_PIXEL_DENSITY;
#if defined(__APPLE__)
    flags |= SDL_WINDOW_METAL;
#endif

    SDL_Window* window = SDL_CreateWindow(
        "MiniEngine", 1280, 720, flags);
    if (window == nullptr)
    {
        std::cerr << "Window creation failed: "
                  << SDL_GetError() << '\n';
        SDL_Quit();
        return 1;
    }

    Renderer renderer;
    if (!renderer.Initialize(window))
    {
        std::cerr << "Renderer initialization failed\n";
        renderer.Shutdown();
        SDL_DestroyWindow(window);
        SDL_Quit();
        return 1;
    }

    bool running = true;
    while (running)
    {
        SDL_Event event;
        while (SDL_PollEvent(&event))
        {
            if (event.type == SDL_EVENT_QUIT ||
                event.type == SDL_EVENT_WINDOW_CLOSE_REQUESTED)
            {
                running = false;
            }
        }

        // Lesson 40 submits and presents a frame here.
    }

    renderer.Shutdown();       // destroys the SDL Metal view on macOS
    SDL_DestroyWindow(window); // must come after renderer shutdown
    SDL_Quit();
    return 0;
}

Windows path — Direct3D 11

SDL owns the Win32 window. The renderer reads SDL's read-only native-handle property, then gives that HWND to DXGI as the swap chain's output window.

#pragma once

struct SDL_Window;
struct ID3D11Device;
struct ID3D11DeviceContext;
struct ID3D11RenderTargetView;
struct IDXGISwapChain;

class Renderer
{
public:
    bool Initialize(SDL_Window* window);
    void Shutdown();

private:
    bool CreateRenderTarget();

    ID3D11Device* device = nullptr;
    ID3D11DeviceContext* context = nullptr;
    IDXGISwapChain* swapChain = nullptr;
    ID3D11RenderTargetView* renderTargetView = nullptr;
};
#include "D3D11Renderer.h"

#include <SDL3/SDL.h>
#include <Windows.h>
#include <d3d11.h>
#include <dxgi.h>

template<typename T>
void SafeRelease(T*& object)
{
    if (object != nullptr)
    {
        object->Release();
        object = nullptr;
    }
}

bool Renderer::CreateRenderTarget()
{
    ID3D11Texture2D* backBuffer = nullptr;
    const HRESULT result = swapChain->GetBuffer(
        0, __uuidof(ID3D11Texture2D),
        reinterpret_cast<void**>(&backBuffer));

    if (FAILED(result))
        return false;

    const HRESULT viewResult = device->CreateRenderTargetView(
        backBuffer, nullptr, &renderTargetView);
    backBuffer->Release();
    return SUCCEEDED(viewResult);
}

bool Renderer::Initialize(SDL_Window* window)
{
    const SDL_PropertiesID properties =
        SDL_GetWindowProperties(window);

    HWND hwnd = static_cast<HWND>(SDL_GetPointerProperty(
        properties,
        SDL_PROP_WINDOW_WIN32_HWND_POINTER,
        nullptr));

    if (hwnd == nullptr)
        return false;

    int pixelWidth = 0;
    int pixelHeight = 0;
    if (!SDL_GetWindowSizeInPixels(
            window, &pixelWidth, &pixelHeight))
        return false;

    DXGI_SWAP_CHAIN_DESC description{};
    description.BufferDesc.Width = static_cast<UINT>(pixelWidth);
    description.BufferDesc.Height = static_cast<UINT>(pixelHeight);
    description.BufferDesc.Format = DXGI_FORMAT_R8G8B8A8_UNORM;
    description.SampleDesc.Count = 1;
    description.BufferUsage = DXGI_USAGE_RENDER_TARGET_OUTPUT;
    description.BufferCount = 2;
    description.OutputWindow = hwnd;
    description.Windowed = TRUE;
    description.SwapEffect = DXGI_SWAP_EFFECT_DISCARD;

    UINT creationFlags = 0;
#if !defined(NDEBUG)
    creationFlags |= D3D11_CREATE_DEVICE_DEBUG;
#endif

    const HRESULT result = D3D11CreateDeviceAndSwapChain(
        nullptr,
        D3D_DRIVER_TYPE_HARDWARE,
        nullptr,
        creationFlags,
        nullptr,
        0,
        D3D11_SDK_VERSION,
        &description,
        &swapChain,
        &device,
        nullptr,
        &context);

    if (FAILED(result) || !CreateRenderTarget())
    {
        Shutdown();
        return false;
    }

    return true;
}

void Renderer::Shutdown()
{
    if (context != nullptr)
    {
        // Drop any back-buffer bindings retained by the immediate context.
        context->ClearState();
        context->Flush();
    }

    SafeRelease(renderTargetView);
    SafeRelease(swapChain);
    SafeRelease(context);
    SafeRelease(device);
}

The swap chain owns the presentable buffers. The render-target view describes how D3D11 will write to its current back buffer. Lesson 40 binds, clears, and presents that view.

macOS path — Metal through SDL3

SDL_Metal_CreateView creates and attaches a Metal-compatible native view. SDL_Metal_GetLayer returns its CAMetalLayer. SDL owns that layer; the renderer borrows the pointer and supplies the Metal device and pixel format.

#pragma once

#include <SDL3/SDL_metal.h>

struct SDL_Window;
namespace CA { class MetalLayer; }
namespace MTL { class CommandQueue; class Device; }

class Renderer
{
public:
    bool Initialize(SDL_Window* window);
    void Shutdown();

private:
    SDL_MetalView metalView = nullptr; // owned by this renderer
    CA::MetalLayer* layer = nullptr;   // borrowed from metalView
    MTL::Device* device = nullptr;
    MTL::CommandQueue* commandQueue = nullptr;
};
#include "MetalRenderer.h"

#include <SDL3/SDL.h>
#include <Metal/Metal.hpp>
#include <QuartzCore/QuartzCore.hpp>

bool Renderer::Initialize(SDL_Window* window)
{
    device = MTL::CreateSystemDefaultDevice();
    if (device == nullptr)
        return false;

    commandQueue = device->newCommandQueue();
    if (commandQueue == nullptr)
    {
        Shutdown();
        return false;
    }

    metalView = SDL_Metal_CreateView(window);
    if (metalView == nullptr)
    {
        Shutdown();
        return false;
    }

    layer = static_cast<CA::MetalLayer*>(
        SDL_Metal_GetLayer(metalView));
    if (layer == nullptr)
    {
        Shutdown();
        return false;
    }

    layer->setDevice(device);
    layer->setPixelFormat(MTL::PixelFormatBGRA8Unorm);
    layer->setFramebufferOnly(true);
    return true;
}

void Renderer::Shutdown()
{
    // SDL owns the layer. Never call layer->release().
    layer = nullptr;

    if (metalView != nullptr)
    {
        SDL_Metal_DestroyView(metalView);
        metalView = nullptr;
    }

    if (commandQueue != nullptr)
    {
        commandQueue->release();
        commandQueue = nullptr;
    }

    if (device != nullptr)
    {
        device->release();
        device = nullptr;
    }
}
Two different ownership rules

The renderer owns SDL_MetalView and must destroy it before SDL_DestroyWindow. The renderer does not own the layer returned by SDL_Metal_GetLayer. Releasing that borrowed pointer causes a lifetime bug.

Checkpoint and common errors

Reload CMake, build, and run. The window may still show an undefined or system-provided background because this lesson initializes presentation but deliberately submits no frame. It should remain responsive, close cleanly, and print no renderer-initialization error. Lesson 40 makes the result visible with a clear color.

SymptomFix
SDL3/SDL.h not foundConfirm FetchContent_MakeAvailable(SDL3), link SDL3::SDL3, then reload CMake.
SDL clone/configure failsCheck internet access and the exact release-3.4.12 tag; delete only the failed CMake build directory and configure again.
Metal/Metal.hpp not foundClone metal-cpp so external/metal-cpp/Metal/Metal.hpp exists, then reload CMake.
Undefined metal-cpp private symbolsAdd MetalCpp.cpp to the target and define each private-implementation macro exactly once.
SDL_Metal_CreateView failsCreate the macOS window with SDL_WINDOW_METAL and call all SDL window/Metal-view functions on the main thread.
D3D creation fails only in DebugInstall Windows Graphics Tools, or temporarily omit D3D11_CREATE_DEVICE_DEBUG.
Recap
  • SDL3 unifies the window and event layer, not Direct3D 11 and Metal commands.
  • Windows obtains SDL's HWND property and builds a D3D11 device, context, swap chain, and render-target view.
  • macOS asks SDL for a Metal view, borrows its layer, and assigns a Metal device and command queue.
  • The SDL Metal view replaces the old hand-written Cocoa/Objective-C++ bridge.
  • Renderer shutdown must finish before the SDL window is destroyed.
