The method names express engine work. No Direct3D or Metal type leaks into the shared application loop.

Windows — resize, clear, and present

The D3D11 render-target view refers to the swap chain's current back buffer. Before resizing the buffers, unbind and release that view; afterward, recreate it with the CreateRenderTarget helper from Lesson 39.

bool Renderer::Resize(int pixelWidth, int pixelHeight)
{
    if (pixelWidth <= 0 || pixelHeight <= 0)
        return true;

    context->OMSetRenderTargets(0, nullptr, nullptr);
    SafeRelease(renderTargetView);

    const HRESULT result = swapChain->ResizeBuffers(
        0,
        static_cast<UINT>(pixelWidth),
        static_cast<UINT>(pixelHeight),
        DXGI_FORMAT_UNKNOWN,
        0);

    if (FAILED(result))
        return false;

    return CreateRenderTarget();
}

void Renderer::RenderFrame()
{
    constexpr float clearColor[4] =
        { 0.04f, 0.08f, 0.16f, 1.0f };

    context->OMSetRenderTargets(
        1, &renderTargetView, nullptr);
    context->ClearRenderTargetView(
        renderTargetView, clearColor);

    // 1 waits for vertical sync and provides basic frame pacing.
    swapChain->Present(1, 0);
}
Why ResizeBuffers can fail

DXGI cannot replace a buffer while your code or the device context still references it. Unbind the old target and release the render-target view before ResizeBuffers. The local back-buffer pointer in CreateRenderTarget is already released in Lesson 39.

macOS — resize, encode, and present

The CAMetalLayer borrowed from the SDL Metal view provides a fresh drawable for each frame. SDL3 automatically updates that layer's drawable size when the window's pixel size changes. The Metal implementation still accepts the shared Resize call, but SDL remains the single owner of layer sizing.

Add the methods below to the same ordinary C++ file, MetalRenderer.cpp, created in Lesson 39:

#include <Foundation/Foundation.hpp>

bool Renderer::Resize(int pixelWidth, int pixelHeight)
{
    if (pixelWidth <= 0 || pixelHeight <= 0)
        return true;

    // SDL3 updates the SDL-owned Metal layer automatically. This backend
    // only confirms that its presentation surface is still available.
    return layer != nullptr;
}

void Renderer::RenderFrame()
{
    // Metal creates autoreleased temporary objects every frame.
    NS::AutoreleasePool* pool =
        NS::AutoreleasePool::alloc()->init();

    CA::MetalDrawable* drawable = layer->nextDrawable();
    if (drawable == nullptr)
    {
        pool->release();
        return; // normal while minimized or temporarily unavailable
    }

    MTL::CommandBuffer* commandBuffer =
        commandQueue->commandBuffer();
    if (commandBuffer == nullptr)
    {
        pool->release();
        return;
    }

    MTL::RenderPassDescriptor* pass =
        MTL::RenderPassDescriptor::renderPassDescriptor();

    MTL::RenderPassColorAttachmentDescriptor* color =
        pass->colorAttachments()->object(0);
    color->setTexture(drawable->texture());
    color->setLoadAction(MTL::LoadActionClear);
    color->setStoreAction(MTL::StoreActionStore);
    color->setClearColor(MTL::ClearColor(
        0.04, 0.08, 0.16, 1.0));

    MTL::RenderCommandEncoder* encoder =
        commandBuffer->renderCommandEncoder(pass);
    if (encoder == nullptr)
    {
        pool->release();
        return;
    }

    encoder->endEncoding();

    commandBuffer->presentDrawable(drawable);
    commandBuffer->commit();
    pool->release();
}

The empty encoder is intentional. The render pass's load action clears the whole color attachment before any geometry exists. Lesson 41 adds a pipeline and draw call.

The layer is still borrowed

These methods use the layer pointer from Lesson 39 but do not change its ownership. Shutdown still destroys metalView and never calls release on the layer itself.

Update the shared SDL3 loop

Replace the placeholder loop from Lesson 39 with this one. SDL supplies physical-pixel dimensions in the resize event, so the application does not need separate Windows and Retina branches.

int pixelWidth = 0;
int pixelHeight = 0;
if (!SDL_GetWindowSizeInPixels(
        window, &pixelWidth, &pixelHeight))
{
    std::cerr << "Pixel-size query failed: "
              << SDL_GetError() << '\n';
    renderer.Shutdown();
    SDL_DestroyWindow(window);
    SDL_Quit();
    return 1;
}

bool running = true;
bool canRender = pixelWidth > 0 && pixelHeight > 0;

// Apply the current size once; later events handle every change.
if (canRender && !renderer.Resize(pixelWidth, pixelHeight))
{
    renderer.Shutdown();
    SDL_DestroyWindow(window);
    SDL_Quit();
    return 1;
}

while (running)
{
    SDL_Event event;
    while (SDL_PollEvent(&event))
    {
        if (event.type == SDL_EVENT_QUIT ||
            event.type == SDL_EVENT_WINDOW_CLOSE_REQUESTED)
        {
            running = false;
        }
        else if (event.type == SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED)
        {
            pixelWidth = event.window.data1;
            pixelHeight = event.window.data2;
            canRender = pixelWidth > 0 && pixelHeight > 0;

            if (canRender &&
                !renderer.Resize(pixelWidth, pixelHeight))
            {
                running = false;
            }
        }
    }

    if (running && canRender)
        renderer.RenderFrame();
}

Keep the shutdown immediately after this loop:

renderer.Shutdown();
SDL_DestroyWindow(window);
SDL_Quit();

This order is required on macOS because Renderer::Shutdown destroys the SDL Metal view attached to the still-living window. It is also a clean ownership order on Windows.

Checkpoint and troubleshooting

Build and run from CLion. The whole drawable area should remain dark blue on Windows or macOS while you move and resize the window. Closing it should exit with code 0.

SymptomLikely fix
D3D window clears once, then fails after resizingUnbind and release the old view before ResizeBuffers, then call CreateRenderTarget.
D3D clear crashes after a failed resizeCheck the Resize return value as shown and stop rendering when recreation fails.
Metal window stays blankKeep the code in MetalRenderer.cpp; confirm SDL_WINDOW_METAL, a non-null drawable, endEncoding, and commit.
Metal memory grows every frameKeep the per-frame NS::AutoreleasePool around drawable, pass, encoder, and command-buffer work.
Retina output is soft or partly filledUse SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED data, not logical values from SDL_GetWindowSize.
Minimizing causes errorsKeep canRender false while either drawable dimension is zero.
