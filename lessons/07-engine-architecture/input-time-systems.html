Input & Time as Engine Services

Our SDL3 window already behaves the same on Windows and macOS. Now we will give gameplay the same portability: an Input service receives SDL events and translates them into engine-owned keys such as Key::W and Key::Space, while a Time service reports how much time passed between frames. Gameplay will use neither Win32, Cocoa nor SDL names.

What you'll learn
  • Why gameplay should use an engine Key enum instead of SDL scancodes
  • How one Input service consumes SDL_Event on every desktop platform
  • How pressed, held and released states live for exactly the intended number of frames
  • Why BeginFrame(), event polling and gameplay queries must happen in that order
  • How std::chrono::steady_clock provides portable delta and total time

SDL owns the platform translation

Windows delivers keyboard and mouse activity through Win32. macOS delivers it through Cocoa. SDL already translates both systems into the same SDL_Event structures, so our engine does not need a Windows input implementation and a separate macOS input implementation.

SDL is still a platform detail, however. If a player controller checks SDL_SCANCODE_W, that controller now depends on SDL. Instead, only Input.cpp knows how SDL scancodes map to the engine's small vocabulary. Gameplay asks about Key::W; a future platform layer or test double can provide that key without changing gameplay.

Give the engine its own key vocabulary

Start with only the keys the engine currently needs. enum class prevents accidental conversion to an integer, and Count gives our fixed arrays their size. We also define engine mouse buttons for the same reason.

#pragma once
#include <array>
#include <cstddef>
#include <cstdint>

// SDL_Event is a union. Forward declaration keeps SDL headers out of
// every gameplay file that includes Input.h.
union SDL_Event;

enum class Key : std::uint8_t {
    W, A, S, D,
    Up, Down, Left, Right,
    Space,
    Escape,
    Count
};

enum class MouseButton : std::uint8_t {
    Left,
    Middle,
    Right,
    Count
};

class Input {
public:
    // Call once at the start of every frame, before SDL_PollEvent().
    void BeginFrame();

    // The engine/platform loop passes every SDL event here.
    // Gameplay never calls this and never receives SDL_Event.
    void ProcessEvent(const SDL_Event& event);

    bool IsKeyDown(Key key) const {
        return m_keysDown[Index(key)];
    }

    bool IsKeyPressed(Key key) const {
        return m_keysPressed[Index(key)];
    }

    bool IsKeyReleased(Key key) const {
        return m_keysReleased[Index(key)];
    }

    bool IsMouseButtonDown(MouseButton button) const {
        return m_mouseButtonsDown[Index(button)];
    }

    float MouseDeltaX() const { return m_mouseDeltaX; }
    float MouseDeltaY() const { return m_mouseDeltaY; }

private:
    static constexpr std::size_t KeyCount =
        static_cast<std::size_t>(Key::Count);
    static constexpr std::size_t MouseButtonCount =
        static_cast<std::size_t>(MouseButton::Count);

    static constexpr std::size_t Index(Key key) {
        return static_cast<std::size_t>(key);
    }

    static constexpr std::size_t Index(MouseButton button) {
        return static_cast<std::size_t>(button);
    }

    std::array<bool, KeyCount> m_keysDown{};
    std::array<bool, KeyCount> m_keysPressed{};
    std::array<bool, KeyCount> m_keysReleased{};
    std::array<bool, MouseButtonCount> m_mouseButtonsDown{};

    float m_mouseDeltaX = 0.0f;
    float m_mouseDeltaY = 0.0f;
};

Pressed, held and released have different lifetimes

A held state survives across frames until a key-up event arrives. Pressed and released are edges: they are cleared at the beginning of the next frame. That gives each query a precise meaning:

QueryWhen it is trueTypical use
IsKeyDownEvery frame from key down until key upWalk while W is held
IsKeyPressedOnly the frame that receives the first key-down transitionJump or open a menu once
IsKeyReleasedOnly the frame that receives the key-up transitionRelease a charged action

Separate edge arrays are slightly more robust than comparing only current and previous snapshots. If a very quick tap produces both key-down and key-up while we drain one frame's event queue, Pressed and Released can both be true for that frame while Down ends false. The tap is not lost.

Translate SDL inside Input.cpp

The mapping helpers live in the implementation file, so no SDL key name escapes into gameplay. SDL scancodes describe a key's physical position, which is what movement controls usually want. Text entry is different: names, chat and console commands should later use SDL_EVENT_TEXT_INPUT, not turn scancodes into characters.

#include "Input.h"

#include <SDL3/SDL.h>
#include <optional>

namespace {

std::optional<Key> ToEngineKey(SDL_Scancode scancode) {
    switch (scancode) {
    case SDL_SCANCODE_W:      return Key::W;
    case SDL_SCANCODE_A:      return Key::A;
    case SDL_SCANCODE_S:      return Key::S;
    case SDL_SCANCODE_D:      return Key::D;
    case SDL_SCANCODE_UP:     return Key::Up;
    case SDL_SCANCODE_DOWN:   return Key::Down;
    case SDL_SCANCODE_LEFT:   return Key::Left;
    case SDL_SCANCODE_RIGHT:  return Key::Right;
    case SDL_SCANCODE_SPACE:  return Key::Space;
    case SDL_SCANCODE_ESCAPE: return Key::Escape;
    default:                   return std::nullopt;
    }
}

std::optional<MouseButton> ToEngineMouseButton(Uint8 button) {
    switch (button) {
    case SDL_BUTTON_LEFT:   return MouseButton::Left;
    case SDL_BUTTON_MIDDLE: return MouseButton::Middle;
    case SDL_BUTTON_RIGHT:  return MouseButton::Right;
    default:                return std::nullopt;
    }
}

} // namespace

void Input::BeginFrame() {
    // Edges and relative motion last for one frame only.
    m_keysPressed.fill(false);
    m_keysReleased.fill(false);
    m_mouseDeltaX = 0.0f;
    m_mouseDeltaY = 0.0f;
}

void Input::ProcessEvent(const SDL_Event& event) {
    switch (event.type) {
    case SDL_EVENT_KEY_DOWN:
    case SDL_EVENT_KEY_UP: {
        const auto key = ToEngineKey(event.key.scancode);
        if (!key) {
            break; // This engine key is not mapped yet.
        }

        const std::size_t index = Index(*key);
        const bool isDown = event.type == SDL_EVENT_KEY_DOWN;

        if (isDown && !m_keysDown[index]) {
            m_keysPressed[index] = true;
        } else if (!isDown && m_keysDown[index]) {
            m_keysReleased[index] = true;
        }

        // Repeated key-down events keep Down true but do not create
        // another Pressed edge because m_keysDown is already true.
        m_keysDown[index] = isDown;
        break;
    }

    case SDL_EVENT_MOUSE_MOTION:
        m_mouseDeltaX += event.motion.xrel;
        m_mouseDeltaY += event.motion.yrel;
        break;

    case SDL_EVENT_MOUSE_BUTTON_DOWN:
    case SDL_EVENT_MOUSE_BUTTON_UP: {
        const auto button = ToEngineMouseButton(event.button.button);
        if (button) {
            m_mouseButtonsDown[Index(*button)] =
                event.type == SDL_EVENT_MOUSE_BUTTON_DOWN;
        }
        break;
    }

    case SDL_EVENT_WINDOW_FOCUS_LOST:
        // Do not leave movement stuck if the player Alt-Tabs on Windows
        // or switches applications with Command-Tab on macOS.
        for (std::size_t i = 0; i < KeyCount; ++i) {
            if (m_keysDown[i]) {
                m_keysReleased[i] = true;
            }
            m_keysDown[i] = false;
        }
        m_mouseButtonsDown.fill(false);
        break;

    default:
        break; // Quit, resize and other events belong to other systems.
    }
}
Key repeat is not another press

Holding a key can produce repeated SDL key-down events for text editing. The transition check !m_keysDown[index] makes IsKeyPressed true only for the first down event. Held movement still works because IsKeyDown remains true.

One loop on Windows and macOS

The engine loop owns SDL event polling. It gives every event to Input, then routes non-input events to the systems that own them. Gameplay runs only after the queue is drained, so it observes the complete input state for this frame.

Input input;
Time time;
bool running = true;

while (running) {
    input.BeginFrame();                    // 1. clear last frame's edges

    SDL_Event event;
    while (SDL_PollEvent(&event)) {        // 2. drain the whole event queue
        input.ProcessEvent(event);         // 3. update this frame's input

        if (event.type == SDL_EVENT_QUIT ||
            event.type == SDL_EVENT_WINDOW_CLOSE_REQUESTED) {
            running = false;
        }

        if (event.type == SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED) {
            renderer.Resize(event.window.data1, event.window.data2);
        }
    }

    time.Tick();                           // 4. update the engine clock
    const float dt = time.DeltaTime();

    // 5. Gameplay uses only engine concepts.
    if (input.IsKeyPressed(Key::Space)) {
        Jump();
    }

    if (input.IsKeyDown(Key::W)) {
        MoveForward(5.0f * dt);
    }

    renderer.RenderFrame();                // 6. draw the updated world
}
The order is part of the contract

Call BeginFrame() before SDL_PollEvent(), and query input only after polling. Calling it after polling erases the new pressed and released edges. Querying during polling gives systems a partial frame whose result depends on event order.

The native source differs, but the engine's frame lifecycle does not. This is the useful boundary: SDL handles platform event delivery, Input owns translation and state, and gameplay owns behavior.

The portable Time service

Almost every system needs to know how much real time elapsed since the last frame — that is delta time. Multiply movement by delta time and your game runs at the same speed whether the machine renders 60 or 240 frames per second. Standard C++ already provides the right portable clock, so this service needs neither SDL nor operating-system code.

#pragma once
#include <chrono>
#include <cstdint>

// Owns the engine clock. Call Tick() once per frame; everyone else
// reads DeltaTime(), TotalTime() or FrameCount().
class Time {
public:
    Time() {
        const auto now = Clock::now();
        m_start = now;
        m_lastFrame = now;
    }

    void Tick() {
        const auto now = Clock::now();
        m_deltaTime =
            std::chrono::duration<float>(now - m_lastFrame).count();
        m_totalTime =
            std::chrono::duration<float>(now - m_start).count();
        m_lastFrame = now;
        ++m_frameCount;
    }

    float DeltaTime() const { return m_deltaTime; }
    float TotalTime() const { return m_totalTime; }
    std::uint64_t FrameCount() const { return m_frameCount; }

private:
    using Clock = std::chrono::steady_clock;

    Clock::time_point m_start;
    Clock::time_point m_lastFrame;
    float m_deltaTime = 0.0f;
    float m_totalTime = 0.0f;
    std::uint64_t m_frameCount = 0;
};
Why steady_clock?

steady_clock is monotonic: it cannot jump backwards when the user changes the system clock or a network time synchronization occurs. Negative frame time would break movement, animation and simulation.

Keep platform names behind the boundary

Gameplay must not use SDL_SCANCODE_W, SDL_GetKeyboardState, GetAsyncKeyState or Cocoa event objects. Add a value to Key and map it once in Input.cpp. That rule keeps the rest of the engine portable and makes input easy to fake in tests.

Recap
  • SDL3 normalizes Win32 and Cocoa events, while the engine-owned Key enum keeps SDL out of gameplay.
  • Input::ProcessEvent is the only place that maps SDL scancodes and updates input state.
  • Held state persists; pressed and released edges last one frame and are cleared by BeginFrame().
  • The frame order is BeginFrame() → drain and route SDL events → Time::Tick() → gameplay update → render.
  • SDL_EVENT_WINDOW_FOCUS_LOST clears held controls on both Windows and macOS so keys cannot become stuck.
  • Time uses portable std::chrono::steady_clock for delta time, total time and frame count.
